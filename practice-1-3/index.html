1.3 · Defining New Functions — Coding Practice

◷ approx. 10 min read · 31 worked examples

These problems drill the core moves of Section 1.3: writing def statements, binding arguments to formal parameters, introducing local names inside a body, treating operators as ordinary functions (add, sub, mul, truediv, floordiv), and composing several small functions into a larger one. No conditionals, loops, or recursion are needed — every solution is a short composition of calls.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Easy

1. Square and Cube

Define square(x) that returns x times itself, then define cube(x) that returns x cubed by reusing square inside its body. The point is that once a function has a name, you can call it from inside another definition just like you call mul.

Examples

Input:  square(5)
Output: 25
Explanation: 5 * 5 = 25.
Input:  cube(2)
Output: 8
Explanation: square(2) = 4, then 4 * 2 = 8.
Input:  cube(square(2))
Output: 64
Explanation: square(2) = 4 first, then cube(4) = 64.

Constraints

  • x is any number.
  • Build cube out of square; do not write x * x * x directly.
  • Use the mul operator function for the multiplications.

Starter

def square(x):
    return mul(x, x)

def cube(x):
    pass
Hint

cube(x) is square(x) multiplied by one more x.

Solutions

Setup

Every solution below assumes these imports and this float-comparison helper are already in scope (the harness defines them once at the top):

from operator import add, sub, mul, truediv, floordiv
from math import sqrt

def approx_eq(x, y, tolerance=1e-9):
    return abs(x - y) < tolerance

1. Square and Cube

Approach — build a function out of another function. Section 1.3 shows that once square has a name, you can call it inside a new definition exactly the way you call mul. cube(x) is square(x) (which is x twice) multiplied by one more x.

def square(x):
    return mul(x, x)

def cube(x):
    return mul(square(x), x)

Why it works. square(x) returns x * x; multiplying that by x gives x * x * x. The composition cube(square(2)) first squares 2 to 4, then cubes 4, evaluating inside-out just like nested call expressions in the lesson.

Complexity. Time O(1), Space O(1). A fixed number of multiplications and one extra local frame per call.

assert square(5) == 25
assert square(-3) == 9
assert cube(2) == 8
assert cube(4) == 64
assert cube(square(2)) == 64

2. Sum of Squares

Approach — compose square inside a two-parameter function. This is the lesson's own sum_squares example: each square(...) opens its own local frame, runs, and returns, and the two returned values become the arguments to add.

def sum_of_squares(x, y):
    return add(square(x), square(y))

Why it works. Both operands of add must be evaluated before add runs, so Python computes square(x) and square(y) first, then adds them. With x = 3, y = 4 that is 9 + 16 = 25.

Complexity. Time O(1), Space O(1). Two square frames open and close in sequence inside one call.

assert sum_of_squares(3, 4) == 25
assert sum_of_squares(0, 7) == 49
assert sum_of_squares(5, 12) == 169
assert sum_of_squares(-2, -2) == 8

3. Average of Two

Approach — true division returns a float. The lesson stresses that / (and its function form truediv) always hands back a float, even when the division comes out even. Adding first and dividing by 2 gives the mean.

def average(a, b):
    return truediv(add(a, b), 2)

Why it works. add(a, b) produces the sum, and truediv(sum, 2) divides it by two as a float — so average(4, 6) is 5.0, not 5.

Complexity. Time O(1), Space O(1). One addition and one division.

assert average(4, 6) == 5.0
assert average(3, 4) == 3.5
assert average(10, 10) == 10.0
assert average(-2, 8) == 3.0

4. Absolute Gap

Approach — operator-as-function plus abs. Subtraction is just the sub function; wrapping its result in abs makes the answer non-negative no matter which argument is larger, so the order of a and b cannot change the result.

def gap(a, b):
    return abs(sub(a, b))

Why it works. sub(a, b) may be positive or negative, but abs collapses both 6 and -6 to 6, so gap(10, 4) and gap(4, 10) agree.

Complexity. Time O(1), Space O(1). One subtraction and one absolute value.

assert gap(10, 4) == 6
assert gap(4, 10) == 6
assert gap(-3, 3) == 6
assert gap(7, 7) == 0

5. Bill With Tip

Approach — a local name inside the body. Straight from the lesson: introduce a local helper gratuity to hold the tip amount, then reuse it in the return expression. gratuity lives only in this call's local frame and goes out of scope when the call returns.

def bill_with_tip(meal_cost, tip_rate):
    gratuity = mul(meal_cost, tip_rate)
    return add(meal_cost, gratuity)

Why it works. gratuity is meal_cost * tip_rate, the dollar value of the tip. Adding it back to meal_cost gives the grand total; because tip_rate is a float, the result is a float (so bill_with_tip(80, 0.2) is 96.0).

Complexity. Time O(1), Space O(1). One extra local binding and a fixed number of operations.

assert approx_eq(bill_with_tip(80, 0.2), 96.0)
assert approx_eq(bill_with_tip(50, 0.18), 59.0)
assert approx_eq(bill_with_tip(120, 0.15), 138.0)
assert approx_eq(bill_with_tip(60, 0.0), 60.0)

6. Distance Between Two Points

Approach — compose sum_of_squares with sqrt. The squared distance is the sum of the squared coordinate differences, which is exactly what sum_of_squares computes; sqrt finishes the job. Reusing a named helper keeps the formula readable.

def distance(x1, y1, x2, y2):
    return sqrt(sum_of_squares(sub(x2, x1), sub(y2, y1)))

Why it works. sub gives the two coordinate differences, sum_of_squares squares and adds them, and sqrt takes the root — the Pythagorean distance. For (0,0) to (3,4) that is sqrt(25) = 5.0.

Complexity. Time O(1), Space O(1). A constant chain of calls, with sum_of_squares opening its own frames inside.

assert approx_eq(distance(0, 0, 3, 4), 5.0)
assert approx_eq(distance(1, 1, 4, 5), 5.0)
assert approx_eq(distance(2, 3, 2, 3), 0.0)
assert approx_eq(distance(-1, -1, 2, 3), 5.0)

7. Clamp Into Range

Approach — compose min and max. No conditionals are available yet, but max and min already encode "at least" and "at most." max(value, low) lifts anything below low up to low; feeding that into min(..., high) caps anything above high.

def clamp(value, low, high):
    return min(max(value, low), high)

Why it works. After max(value, low), the result is never below low; after the surrounding min(..., high), it is never above high; and a value already inside the range passes through both untouched.

Complexity. Time O(1), Space O(1). Two built-in comparisons.

assert clamp(5, 0, 10) == 5
assert clamp(-3, 0, 10) == 0
assert clamp(42, 0, 10) == 10
assert clamp(10, 0, 10) == 10
assert clamp(0, 0, 10) == 0

8. Cents to Dollars and Change

Approach — floor division, then build the second function on the first. whole_dollars uses floordiv to drop to whole dollars. leftover_cents reuses whole_dollars rather than recomputing it, then subtracts the dollar amount (times 100) from the total — composition over duplication.

def whole_dollars(cents):
    return floordiv(cents, 100)

def leftover_cents(cents):
    return sub(cents, mul(whole_dollars(cents), 100))

Why it works. floordiv(cents, 100) is the number of whole dollars; multiplying that back by 100 and subtracting from cents leaves exactly the change. For 1234, that is 1234 - 12 * 100 = 34.

Complexity. Time O(1), Space O(1). A constant number of arithmetic operations per call.

assert whole_dollars(250) == 2
assert leftover_cents(250) == 50
assert whole_dollars(99) == 0
assert leftover_cents(99) == 99
assert whole_dollars(100) == 1
assert leftover_cents(100) == 0
assert leftover_cents(1234) == 34

9. Quadratic Mean

Approach — layered composition of named helpers. The root-mean-square is sqrt(average(square(x), square(y))). Splitting it into mean_square (the average of the squares) and quadratic_mean (its root) means each function carries one clear idea, and each reuses square, average, and sqrt from earlier.

def mean_square(x, y):
    return average(square(x), square(y))

def quadratic_mean(x, y):
    return sqrt(mean_square(x, y))

Why it works. mean_square(3, 4) averages 9 and 16 to 12.5; quadratic_mean takes sqrt(12.5), which is 3.5355339059327378. Each layer is a single, verifiable step.

Complexity. Time O(1), Space O(1). A fixed chain of calls; several small frames open and close.

assert mean_square(3, 4) == 12.5
assert approx_eq(quadratic_mean(3, 4), 3.5355339059327378)
assert approx_eq(quadratic_mean(0, 0), 0.0)
assert approx_eq(quadratic_mean(6, 8), 7.0710678118654755)

10. Larger Root of a Quadratic

Approach — name the discriminant, then compose. Splitting out discriminant(s, p) (which reuses square for s2) keeps larger_root short and readable, then larger_root plugs that into the + branch of the quadratic formula and divides by 2.

def discriminant(s, p):
    return sub(square(s), mul(4, p))

def larger_root(s, p):
    return truediv(add(s, sqrt(discriminant(s, p))), 2)

Why it works. The roots of x2−sx+p are (s±s2−4p​)/2; the larger one uses +. For s = 5, p = 6, the discriminant is 1, and (5 + 1) / 2 = 3.0. A double root (discriminant 0) returns the single value, as with larger_root(4, 4) = 2.0.

Complexity. Time O(1), Space O(1). A constant chain of calls plus one sqrt.

assert approx_eq(larger_root(5, 6), 3.0)
assert approx_eq(larger_root(7, 12), 4.0)
assert approx_eq(larger_root(4, 4), 2.0)
