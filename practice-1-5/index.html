Solutions

1. Sign of a Number

Approach — three-way if/elif/else. This mirrors absolute_value from 1.5-A: Python checks the clauses top to bottom and runs the suite of the first true one, skipping the rest. Exactly one of the three branches fires.

def sign(x):
    if x > 0:
        return 1
    elif x == 0:
        return 0
    else:
        return -1

Why it works. x > 0 catches every positive value; if that is false, x == 0 catches exactly zero; if both are false, x must be negative, so the else returns -1.

Complexity. Time O(1), Space O(1). A fixed chain of comparisons with no loop.

assert sign(7) == 1
assert sign(0) == 0
assert sign(-3) == -1
assert sign(0.0001) == 1

2. Clamp to a Range

Approach — ordered conditional decisions. Three exclusive cases — below the floor, above the ceiling, or already inside — map directly onto an if/elif/else. Because only the first true clause runs, the inside case never needs to re-test the bounds.

def clamp(x, low, high):
    if x < low:
        return low
    elif x > high:
        return high
    else:
        return x

Why it works. If x is below low it snaps up; if it is above high it snaps down; otherwise it is within [low, high] and passes through. The boundary values satisfy neither strict inequality, so they fall into the else and return unchanged.

Complexity. Time O(1), Space O(1). Two comparisons at most.

assert clamp(5, 0, 10) == 5
assert clamp(-4, 0, 10) == 0
assert clamp(99, 0, 10) == 10
assert clamp(10, 0, 10) == 10
assert clamp(0, 0, 10) == 0

3. Count Multiples up to n

Approach — counter loop with a divisibility test. A while loop walks k from 1 to n. Inside, the remainder expression k - (k // d) * d (built from floor division, since % is not in this section) is 0 exactly when d divides k, and the counter rises on those passes. The d == 0 guard runs first so we never floor-divide by zero.

def count_multiples(n, d):
    if d == 0:
        return 0
    count, k = 0, 1
    while k <= n:
        if k - (k // d) * d == 0:
            count = count + 1
        k = k + 1
    return count

Why it works. Each integer in 1..n is visited once; count increases precisely for the multiples of d. The k = k + 1 is the binding-change that moves the loop toward k > n and stops it.

Complexity. Time O(n), Space O(1). One pass over the counter with constant extra storage.

assert count_multiples(10, 2) == 5
assert count_multiples(10, 3) == 3
assert count_multiples(9, 3) == 3
assert count_multiples(5, 7) == 0
assert count_multiples(6, 0) == 0

4. Safe Reciprocal

Approach — short-circuit guard. This is the x != 0 and ... pattern from 1.5-A. Python evaluates the left operand of and first; when x is 0 the guard is false, so the whole and is false and the right side 1 / x is never evaluated — no division-by-zero ever happens.

def safe_reciprocal(x):
    if x != 0 and 1 / x == 1 / x:
        return 1 / x
    return 0

Why it works. For nonzero x, the guard passes, the harmless 1 / x == 1 / x check is true, and 1 / x is returned. For x == 0, the guard short-circuits and control falls through to return 0, so the risky division is skipped entirely.

Complexity. Time O(1), Space O(1). A constant amount of work either way.

assert safe_reciprocal(2) == 0.5
assert safe_reciprocal(4) == 0.25
assert safe_reciprocal(0) == 0
assert safe_reciprocal(-5) == -0.2

5. Number of Digits

Approach — shrink-by-floor-division loop. After taking abs(n) and handling the special case 0, a while loop repeatedly applies n = n // 10, lopping off one trailing digit each pass, and counts how many passes it takes to reach 0.

def digit_count(n):
    n = abs(n)
    if n == 0:
        return 1
    count = 0
    while n > 0:
        n = n // 10
        count = count + 1
    return count

Why it works. Floor-dividing by 10 removes the last digit, so the number of passes before n becomes 0 is exactly the number of digits. The n // 10 each pass is the binding-change that guarantees the loop ends. Zero is handled up front because the loop would otherwise count it as 0 digits.

Complexity. Time O(logn) in the value of n (one pass per digit), Space O(1).

assert digit_count(0) == 1
assert digit_count(7) == 1
assert digit_count(42) == 2
assert digit_count(1000) == 4
assert digit_count(-583) == 3

6. Greatest Common Divisor

Approach — Euclid by subtraction. The loop condition a > 0 and b > 0 keeps going only while both numbers are positive. Each pass replaces the larger value with their difference, which shrinks the pair while preserving the gcd. When one value hits 0, the other holds the answer, so a + b returns it.

def gcd(a, b):
    a, b = abs(a), abs(b)
    while a > 0 and b > 0:
        if a > b:
            a = a - b
        else:
            b = b - a
    return a + b

Why it works. Subtracting the smaller from the larger never changes the set of common divisors, so the gcd is invariant across passes. The values strictly decrease, so the loop must terminate; at the end exactly one of a, b is 0, and adding them returns the nonzero one (or 0 if both started at 0).

Complexity. Time O(max(a,b)) in the worst case (repeated subtraction), Space O(1).

assert gcd(12, 18) == 6
assert gcd(48, 36) == 12
assert gcd(17, 5) == 1
assert gcd(0, 9) == 9
assert gcd(14, 0) == 14
assert gcd(100, 100) == 100

7. Collatz Step Count

Approach — while loop with an inner branch. The loop runs while n > 1. Each pass uses an if/else to pick the even or odd rule, updates n, and adds one to steps. Evenness is tested with the floor-division remainder n - (n // 2) * 2 == 0 since % is not available yet.

def collatz_steps(n):
    steps = 0
    while n > 1:
        if n - (n // 2) * 2 == 0:
            n = n // 2
        else:
            n = 3 * n + 1
        steps = steps + 1
    return steps

Why it works. Each pass transforms n by exactly one Collatz rule and tallies one step. The loop stops when n reaches 1; for n == 1 the body never runs, so the count is 0.

Complexity. Time O(s) where s is the number of steps to reach 1, Space O(1). The step count is the loop length itself.

assert collatz_steps(1) == 0
assert collatz_steps(2) == 1
assert collatz_steps(6) == 8
assert collatz_steps(7) == 16
assert collatz_steps(27) == 111

8. Primality Test

Approach — early-exit trial division. Numbers below 2 are rejected outright. Then a while loop tries divisors d while d * d <= n; the first d that divides n (again via the floor-division remainder) triggers an immediate return False. Surviving the whole loop means no divisor exists, so n is prime.

def is_prime(n):
    if n < 2:
        return False
    d = 2
    while d * d <= n:
        if n - (n // d) * d == 0:
            return False
        d = d + 1
    return True

Why it works. Any composite n has a divisor no larger than its square root, so checking d up to d * d <= n is sufficient. Returning False at the first divisor short-circuits the search; reaching the end with none found proves primality.

Complexity. Time O(n​), Space O(1). The loop runs only while d * d <= n.

assert is_prime(2) is True
assert is_prime(3) is True
assert is_prime(4) is False
assert is_prime(1) is False
assert is_prime(0) is False
assert is_prime(97) is True
assert is_prime(100) is False

9. Reverse the Digits

Approach — peel and rebuild. Record the sign, work with abs(n), and on each pass pull off the lowest digit with the remainder trick while shifting the growing result left by a place. This is a single while loop whose state — the shrinking n and the growing result — moves together until n reaches 0.

def reverse_number(n):
    negative = n < 0
    n = abs(n)
    result = 0
    while n > 0:
        digit = n - (n // 10) * 10
        result = result * 10 + digit
        n = n // 10
    if negative:
        return -result
    return result

Why it works. digit = n - (n // 10) * 10 is the last digit of n; appending it to result with result * 10 + digit places it one position more significant than the previous digit, which reverses the order. Floor-dividing n by 10 drives the loop to termination. Trailing zeros vanish because a leading zero in the reversed number contributes nothing to the integer value.

Complexity. Time O(logn) in the value of n (one pass per digit), Space O(1).

assert reverse_number(0) == 0
assert reverse_number(7) == 7
assert reverse_number(123) == 321
assert reverse_number(1200) == 21
assert reverse_number(-405) == -504

10. The nth Prime

Approach — nested loops with a short-circuited flag. The outer while keeps generating candidates until found reaches n. For each candidate, an inner while does trial division, setting is_p to False at the first divisor; the header d * d <= candidate and is_p short-circuits so the inner loop stops the instant the candidate is known to be composite.

def nth_prime(n):
    found, candidate = 0, 1
    while found < n:
        candidate = candidate + 1
        is_p, d = True, 2
        while d * d <= candidate and is_p:
            if candidate - (candidate // d) * d == 0:
                is_p = False
            d = d + 1
        if is_p:
            found = found + 1
    return candidate

Why it works. candidate starts at 1 and is bumped before each test, so the first candidate examined is 2. The inner loop decides primality exactly as is_prime does; when a candidate survives, found increases. The outer loop exits the moment found == n, leaving candidate bound to the nth prime.

Complexity. Time O(pp​) where p is the value of the nth prime (each candidate up to p costs an O(p​) trial-division), Space O(1).
