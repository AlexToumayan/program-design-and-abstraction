3.2 · Functional Programming in Scheme — Coding Practice

◷ approx. 11 min read · 21 worked examples

These problems build fluency with Scheme's functional core: defining procedures, prefix arithmetic and the if special form, building and taking apart pairs and lists with cons/car/cdr, writing recursive list procedures with a null? base case, passing procedures to higher-order procedures, and using a nested helper under lexical scope.

> Work each problem from the starter signature first. Each problem is posed as a Scheme procedure; write your Scheme solution, run it against the Examples (shown as Scheme calls and their printed values), then open the matching solution below to check your approach and its time and space cost. Every solution also ships an equivalent Python model so the page is machine-verified.

A note on how Scheme lists are modelled here. The Examples show real Scheme call syntax and the value Scheme prints. The verification blocks model Scheme data in Python exactly the way the teaching documents do: nil is None, a pair (cons a b) is the tuple (a, b), car/cdr select the two parts, null? tests for nil, and a symbol such as 'a is the Python string "a". The helper scm(...) models Scheme's list, and py(...) flattens a modelled list back to a plain Python list so the asserts read cleanly. These helpers are shown once in the Setup block at the top of the Solutions.

Easy

1. Cube

Write a Scheme procedure cube that returns its single number argument multiplied by itself three times. Use prefix arithmetic and a one-line define.

Examples

Input:  (cube 2)
Output: 8
Explanation: 2 * 2 * 2 = 8.

Input:  (cube 5)
Output: 125
Explanation: 5 * 5 * 5 = 125.

Input:  (cube -3)
Output: -27
Explanation: (-3) * (-3) * (-3) = -27.

Constraints

  • x is an integer.
  • Use a single define with a prefix-notation body; no if needed.

Starter

(define (cube x)
  ???)
Hint

(* x x) squares; multiply by one more x to cube.

2. Clamp from Below

Write a Scheme procedure clamp-low that takes a number x and a floor lo. If x is below lo, return lo; otherwise return x. The result is never below the floor.

Examples

Input:  (clamp-low 3 10)
Output: 10
Explanation: 3 is below the floor 10, so the floor is returned.

Input:  (clamp-low 42 10)
Output: 42
Explanation: 42 is already at or above 10, so it passes through.

Input:  (clamp-low 10 10)
Output: 10
Explanation: equal to the floor counts as "not below," so x passes through.

Constraints

  • x and lo are integers.
  • Use the if special form with a < comparison as the predicate.

Starter

(define (clamp-low x lo)
  ???)
Hint

The predicate (< x lo) is true exactly when x needs to be pulled up to lo.

3. Third Element

Write a Scheme procedure third that returns the third element of a list (the element at position two, counting from zero) by composing car and cdr. Assume the list has at least three elements.

Examples

Input:  (third (list 10 20 30 40))
Output: 30
Explanation: drop two elements with two cdrs, then car the rest.

Input:  (third (list 'a 'b 'c))
Output: c
Explanation: the third symbol is c.

Input:  (third (list 1 2 3 4 5))
Output: 3
Explanation: index 2 is the value 3.

Constraints

  • items is a list with at least three elements.
  • Use only car and cdr (no recursion, no index helper).

Starter

(define (third items)
  ???)
Hint

(cdr (cdr items)) is the list starting at the third element; car lifts that element out.

4. Sum a List

Write a recursive Scheme procedure sum-list that adds up every number in a list. The empty list sums to 0.

Examples

Input:  (sum-list nil)
Output: 0
Explanation: the empty list is the base case.

Input:  (sum-list (list 1 2 3 4))
Output: 10
Explanation: 1 + 2 + 3 + 4 = 10.

Input:  (sum-list (list 10 20 30))
Output: 60
Explanation: 10 + 20 + 30 = 60.

Constraints

  • items is a list of integers (possibly empty).
  • Use recursion with a (null? items) base case; no loops.

Starter

(define (sum-list items)
  ???)
Hint

Add the first element (car items) to the sum of the rest (sum-list (cdr items)), stopping at nil.

Medium

5. Count Occurrences

Write a recursive Scheme procedure count-occurrences that counts how many times a number x appears in a list items.

Examples

Input:  (count-occurrences 3 (list 3 1 3 3 2))
Output: 3
Explanation: the value 3 appears three times.

Input:  (count-occurrences 7 (list 1 2 3))
Output: 0
Explanation: 7 never appears.

Input:  (count-occurrences 5 (list 5 5 5 5))
Output: 4
Explanation: every element matches.

Constraints

  • items is a list of integers (possibly empty); x is an integer.
  • Use recursion. Add 1 only when (= x (car items)).

Starter

(define (count-occurrences x items)
  ???)
Hint

Each step compares the first element to x, adds 1 if it matches, and recurses on the rest either way.

6. Map a Procedure Over a List

Write a higher-order Scheme procedure my-map that takes a one-argument procedure fn and a list items, and returns a new list where fn has been applied to each element. The original list is untouched.

Examples

Input:  (my-map (lambda (n) (+ n 1)) (list 1 2 3))
Output: (2 3 4)
Explanation: each element is incremented by one.

Input:  (my-map (lambda (n) (* n n)) (list 2 3 4))
Output: (4 9 16)
Explanation: each element is squared.

Input:  (my-map (lambda (n) (+ n 1)) nil)
Output: ()
Explanation: mapping over the empty list gives the empty list.

Constraints

  • fn is a procedure of one argument; items is a list.
  • Build the result with cons; recurse with a (null? items) base case returning nil.

Starter

(define (my-map fn items)
  ???)
Hint

cons the transformed first element (fn (car items)) onto the mapped rest (my-map fn (cdr items)).

7. Append Two Lists

Write a recursive Scheme procedure my-append that joins two lists end to end, returning a single list with every element of a followed by every element of b.

Examples

Input:  (my-append (list 1 2) (list 3 4 5))
Output: (1 2 3 4 5)
Explanation: the elements of the first list come first.

Input:  (my-append nil (list 9 8))
Output: (9 8)
Explanation: appending to an empty front list returns the second list.

Input:  (my-append (list 'a 'b) (list 'c))
Output: (a b c)
Explanation: works on lists of symbols too.

Constraints

  • a and b are lists (either may be empty).
  • Recurse on a. The base case (null? a) returns b; rebuild a element by element with cons.

Starter

(define (my-append a b)
  ???)
Hint

When a is empty the answer is just b; otherwise cons the first of a onto the append of the rest of a with b.

8. Keep Elements That Pass a Test

Write a higher-order Scheme procedure keep that takes a predicate procedure fn and a list items, and returns a new list of only the elements for which (fn element) is true, in their original order.

Examples

Input:  (keep (lambda (n) (> n 0)) (list -2 5 -1 3 0))
Output: (5 3)
Explanation: only the strictly positive elements survive.

Input:  (keep (lambda (n) (= (remainder n 2) 0)) (list 1 2 3 4 5 6))
Output: (2 4 6)
Explanation: only the even elements pass the test.

Input:  (keep (lambda (n) (> n 100)) (list 1 2 3))
Output: ()
Explanation: nothing passes, so the result is empty.

Constraints

  • fn is a one-argument predicate returning a boolean; items is a list.
  • Use recursion; keep an element by cons-ing it on, drop it by recursing without it.

Starter

(define (keep fn items)
  ???)
Hint

At each step, test the first element. If it passes, cons it onto the filtered rest; if not, return just the filtered rest.

Hard

9. Reverse a List

Write a Scheme procedure reverse-list that returns a new list with the elements of items in reverse order. Use a nested helper procedure that carries an accumulator, relying on lexical scope.

Examples

Input:  (reverse-list (list 1 2 3 4))
Output: (4 3 2 1)
Explanation: the order is flipped.

Input:  (reverse-list (list 'a 'b 'c))
Output: (c b a)
Explanation: works on symbols too.

Input:  (reverse-list nil)
Output: ()
Explanation: reversing the empty list gives the empty list.

Constraints

  • items is a list (possibly empty).
  • Define a nested helper inside reverse-list that takes the remaining list and an accumulator; start the accumulator at nil.

Starter

(define (reverse-list items)
  (define (helper items acc)
    ???)
  (helper items nil))
Hint

Each step conses the current first element onto the accumulator, so the front of the input ends up at the back of the result.

10. Insert Into a Sorted List

Write a recursive Scheme procedure insert that places a number x into an already-ascending list items so the result stays in ascending order. If x ties an existing element, it may go on either side.

Examples

Input:  (insert 3 (list 1 2 4 5))
Output: (1 2 3 4 5)
Explanation: 3 slots between 2 and 4.

Input:  (insert 0 (list 1 2 3))
Output: (0 1 2 3)
Explanation: 0 is smaller than everything, so it leads.

Input:  (insert 9 (list 1 2 3))
Output: (1 2 3 9)
Explanation: 9 is larger than everything, so it trails.

Constraints

  • items is a list of integers sorted in ascending order (possibly empty); x is an integer.
  • Use recursion that rebuilds the prefix with cons until x finds its slot.

Starter

(define (insert x items)
  ???)
Hint

Three cases: the list is empty (make a one-element list), x is smaller than the first element (cons it on the front), or x belongs further back (cons the first element onto insert of the rest).

Solutions

Setup

Every verification block below models Scheme data in Python the way the teaching documents do, using these shared definitions. Run this block first in a fresh interpreter; the rest of the page builds on it.

1. Cube

Approach — prefix arithmetic in a one-line define. This is the squaring example from the section ((define (square x) (* x x))) extended by one factor. The body multiplies x by itself three times, all in prefix notation with no if needed.

(define (cube x) (* x x x))

The Python model of the same procedure:

def cube(x):
    return x * x * x

Why it works. A single define binds cube to a procedure whose body is one combination; applying it substitutes the argument for x and evaluates (* x x x).

Complexity. Time O(1), Space O(1) — a fixed amount of arithmetic per call.

assert cube(2) == 8
assert cube(5) == 125
assert cube(0) == 0
assert cube(-3) == -27

2. Clamp from Below

Approach — the if special form with a < predicate. This mirrors the abs procedure from the section, which branches on (< x 0). Here the predicate (< x lo) chooses between the floor and the value itself, and only the selected branch is evaluated.

(define (clamp-low x lo)
  (if (< x lo)
      lo
      x))

The Python model:

def clamp_low(x, lo):
    if x < lo:
        return lo
    else:
        return x

Why it works. Because if is a special form, exactly one branch runs: when x is below the floor the consequent returns lo, otherwise the alternative returns the unchanged x.

Complexity. Time O(1), Space O(1) — one comparison and one branch.

assert clamp_low(3, 10) == 10
assert clamp_low(42, 10) == 42
assert clamp_low(10, 10) == 10
assert clamp_low(-5, 0) == 0

3. Third Element

Approach — compose car and cdr. Two cdrs skip the first two elements, leaving a list whose first element is the one we want; a final car lifts it out. This is the same selector-chaining the section uses for (car (cdr one-through-four)), taken one step deeper.

(define (third items)
  (car (cdr (cdr items))))

The Python model (with car/cdr from the Setup block):

def third(items):
    return car(cdr(cdr(items)))

Why it works. (cdr items) is the list from element two onward, (cdr (cdr items)) is the list from element three onward, and car returns that list's first element, which is the third overall.

Complexity. Time O(1), Space O(1) — a fixed three selector calls regardless of list length.

assert third(scm(10, 20, 30, 40)) == 30
assert third(scm("a", "b", "c")) == "c"
assert third(scm(1, 2, 3, 4, 5)) == 3

4. Sum a List

Approach — recursive list reduction with a null? base case. This follows the standard list-procedure skeleton from the section (the same shape as length): handle the empty list as the base case, otherwise combine the first element with the recursive result on the rest.

(define (sum-list items)
  (if (null? items)
      0
      (+ (car items) (sum-list (cdr items)))))

The Python model:

def sum_list(items):
    if null(items):
        return 0
    else:
        return car(items) + sum_list(cdr(items))

Why it works. The empty list contributes 0; every other call adds its first element to the sum of a strictly shorter list, so the recursion bottoms out at nil.

Complexity. Time O(n), Space O(n) — one frame per element of the list.

assert sum_list(nil) == 0
assert sum_list(scm(4)) == 4
assert sum_list(scm(1, 2, 3, 4)) == 10
assert sum_list(scm(10, 20, 30)) == 60

5. Count Occurrences

Approach — recursion with a nested if and =. Same list skeleton as sum-list, but the combine step is conditional: add 1 only when the first element equals x. The outer if handles the empty list; the inner if decides whether the first element counts.

(define (count-occurrences x items)
  (if (null? items)
      0
      (if (= x (car items))
          (+ 1 (count-occurrences x (cdr items)))
          (count-occurrences x (cdr items)))))

The Python model:

def count_occurrences(x, items):
    if null(items):
        return 0
    else:
        if x == car(items):
            return 1 + count_occurrences(x, cdr(items))
        else:
            return count_occurrences(x, cdr(items))

Why it works. Each element is inspected exactly once; a match adds 1 to the count of the rest, a non-match adds nothing, and the empty list contributes 0.

Complexity. Time O(n), Space O(n) — one frame per element.

assert count_occurrences(3, nil) == 0
assert count_occurrences(3, scm(3, 1, 3, 3, 2)) == 3
assert count_occurrences(7, scm(1, 2, 3)) == 0
assert count_occurrences(5, scm(5, 5, 5, 5)) == 4

6. Map a Procedure Over a List

Approach — higher-order procedure that rebuilds a list. This is the squares skeleton from the section generalized: instead of hard-coding squaring, the transform is passed in as fn. The base case returns nil, and each step conses (fn (car items)) onto the mapped rest.

(define (my-map fn items)
  (if (null? items)
      nil
      (cons (fn (car items))
            (my-map fn (cdr items)))))

The Python model:

def my_map(fn, items):
    if null(items):
        return nil
    else:
        return cons(fn(car(items)), my_map(fn, cdr(items)))

Why it works. A fresh pair is built for every element with the transformed value in front, so the result is a brand-new list of transformed elements; the original list is never mutated.

Complexity. Time O(n), Space O(n) — one new pair and one frame per element.

assert py(my_map(lambda n: n + 1, scm(1, 2, 3))) == [2, 3, 4]
assert py(my_map(lambda n: n * n, scm(2, 3, 4))) == [4, 9, 16]
assert py(my_map(lambda n: n + 1, nil)) == []

7. Append Two Lists

Approach — recurse on the first list, rebuild it onto the second. The base case (null? a) returns b unchanged. Otherwise the answer is the first element of a consed onto the append of the rest of a with b, so a is rebuilt element by element in front of b.

(define (my-append a b)
  (if (null? a)
      b
      (cons (car a) (my-append (cdr a) b))))

The Python model:

def my_append(a, b):
    if null(a):
        return b
    else:
        return cons(car(a), my_append(cdr(a), b))

Why it works. Only a is taken apart and rebuilt; b is shared as the tail. When a runs out, the recursion returns b, and the unwinding conses each element of a back on in order.

Complexity. Time O(n), Space O(n) for n the length of a — one frame and one new pair per element of a.

assert py(my_append(scm(1, 2), scm(3, 4, 5))) == [1, 2, 3, 4, 5]
assert py(my_append(nil, scm(9, 8))) == [9, 8]
assert py(my_append(scm(7), nil)) == [7]
assert py(my_append(scm("a", "b"), scm("c"))) == ["a", "b", "c"]

8. Keep Elements That Pass a Test

Approach — higher-order filter via recursion. Same skeleton as my-map, but the recursive case branches: when (fn (car items)) is true the element is consed onto the filtered rest, and when it is false the element is dropped by returning just the filtered rest.

(define (keep fn items)
  (if (null? items)
      nil
      (if (fn (car items))
          (cons (car items) (keep fn (cdr items)))
          (keep fn (cdr items)))))

The Python model:

def keep(fn, items):
    if null(items):
        return nil
    else:
        if fn(car(items)):
            return cons(car(items), keep(fn, cdr(items)))
        else:
            return keep(fn, cdr(items))

Why it works. Every element is tested once. Passing elements are rebuilt into a fresh list in their original order; failing elements simply never get consed on. The empty list yields the empty list.

Complexity. Time O(n), Space O(n) — one frame per element; the result holds only the kept elements.

assert py(keep(lambda n: n > 0, scm(-2, 5, -1, 3, 0))) == [5, 3]
assert py(keep(lambda n: n % 2 == 0, scm(1, 2, 3, 4, 5, 6))) == [2, 4, 6]
assert py(keep(lambda n: n > 100, scm(1, 2, 3))) == []

9. Reverse a List

Approach — nested helper with an accumulator, under lexical scope. Like the sqrt example from the section, reverse-list defines a helper inside itself. The helper walks items while building up acc, cons-ing each element it sees onto the front of acc; because elements are pushed on as they are visited, the first element ends up deepest, which is the reverse order.

(define (reverse-list items)
  (define (helper items acc)
    (if (null? items)
        acc
        (helper (cdr items) (cons (car items) acc))))
  (helper items nil))

The Python model:

def reverse_list(items):
    def helper(items, acc):
        if null(items):
            return acc
        else:
            return helper(cdr(items), cons(car(items), acc))
    return helper(items, nil)

Why it works. Each step moves one element from the front of the remaining list to the front of the accumulator. The element seen first is buried the deepest, so when the input is exhausted the accumulator holds the elements in reversed order.

Complexity. Time O(n), Space O(n) — one helper call per element; the accumulator grows to length n.

assert py(reverse_list(scm(1, 2, 3, 4))) == [4, 3, 2, 1]
assert py(reverse_list(nil)) == []
assert py(reverse_list(scm("a", "b", "c"))) == ["c", "b", "a"]
assert py(reverse_list(scm(42))) == [42]

10. Insert Into a Sorted List

Approach — recursion that rebuilds the prefix until x fits. Three cases drive the recursion. If the list is empty, x becomes a one-element list. If x is smaller than the first element, it belongs right here, so cons it onto the whole list. Otherwise the first element stays in front and x must be inserted into the rest.

(define (insert x items)
  (if (null? items)
      (cons x nil)
      (if (< x (car items))
          (cons x items)
          (cons (car items) (insert x (cdr items))))))

The Python model:

def insert(x, items):
    if null(items):
        return cons(x, nil)
    else:
        if x < car(items):
            return cons(x, items)
        else:
            return cons(car(items), insert(x, cdr(items)))

Why it works. The procedure walks past every element smaller than x, rebuilding them in front, and stops at the first element that is not smaller, dropping x ahead of it. If no such element exists, x lands at the end via the empty-list base case. Since the input was sorted, the output stays sorted.

Complexity. Time O(n), Space O(n) — in the worst case x is larger than every element, so the whole prefix is rebuilt.

assert py(insert(3, scm(1, 2, 4, 5))) == [1, 2, 3, 4, 5]
assert py(insert(0, scm(1, 2, 3))) == [0, 1, 2, 3]
assert py(insert(9, scm(1, 2, 3))) == [1, 2, 3, 9]
assert py(insert(5, nil)) == [5]
assert py(insert(2, scm(2, 4))) == [2, 2, 4]
