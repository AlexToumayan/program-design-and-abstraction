1.4 · Designing Functions — Coding Practice

◷ approx. 11 min read · 31 worked examples

These problems drill the design tools of Section 1.4: writing a function that keeps one honest promise, documenting it with a docstring, leaving a comment on a single tricky line, and — above all — giving parameters default argument values so the common case is easy to call while a caller can still override it. No conditionals, loops, or recursion are needed; each solution is a short, well-documented arithmetic contract.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Easy

1. Shipping Cost

Define shipping_cost(weight, rate=2.5) that returns the cost to ship a parcel by weight. The standard rate is 2.5 per kilogram, so make that the default argument value for rate: a one-argument call falls back to it, a two-argument call overrides it. Give the function a docstring documenting both parameters.

Examples

Input:  shipping_cost(4)
Output: 10.0
Explanation: rate defaults to 2.5, so 4 * 2.5 = 10.0.
Input:  shipping_cost(10)
Output: 25.0
Explanation: 10 * 2.5 = 25.0, still using the default rate.
Input:  shipping_cost(4, 3.0)
Output: 12.0
Explanation: the supplied 3.0 overrides the default; 4 * 3.0 = 12.0.

Constraints

  • weight is a non-negative number; rate is a number.
  • Make rate an optional parameter with default 2.5 in the header.
  • Write a docstring whose first line states the job in one sentence.

Starter

def shipping_cost(weight, rate=2.5):
    pass
Hint

The = in the header sets a default; it does not run an assignment each call.

2. Final Grade

Define final_grade(homework, exam, exam_weight=0.6) that blends a homework score and an exam score into one grade. The exam is worth 60% by default, so exam_weight=0.6; the homework weight is whatever is left over. Bind that leftover to a local name in the body and comment the line.

Examples

Input:  final_grade(100, 50)
Output: 70.0
Explanation: homework weight is 1 - 0.6 = 0.4, so 100 * 0.4 + 50 * 0.6 = 70.0.
Input:  final_grade(80, 90)
Output: 86.0
Explanation: 80 * 0.4 + 90 * 0.6 = 32 + 54 = 86.0.
Input:  final_grade(80, 90, 0.5)
Output: 85.0
Explanation: with exam_weight 0.5 the two weights are equal: (80 + 90) / 2 = 85.0.

Constraints

  • homework and exam are numbers; exam_weight is a fraction between 0 and 1.
  • Default exam_weight to 0.6.
  • The two weights must always sum to 1: derive the homework weight as 1 - exam_weight.

Starter

def final_grade(homework, exam, exam_weight=0.6):
    pass
Hint

Compute homework_weight = 1 - exam_weight on its own line, then combine the two weighted scores.

Output: 6.283185307179586

  • radius is a non-negative number; scale is a number.
  • Use pi from the math module (shown in Setup); do not hard-code a rounded value.
  • Default scale to 1.

def circle_area(radius, scale=1):

Hint

pi is a fixed constant in the body; scale is the thing a caller might want to change, so it earns a default.

5. Future Balance

Define future_balance(principal, rate=0.05, years=1) that applies simple interest and returns the new balance. The standard rate is 5% and the common horizon is one year, so both get defaults. Introduce a local name interest for the amount earned and comment it.

Examples

Input:  future_balance(1000)
Output: 1050.0
Explanation: interest = 1000 * 0.05 * 1 = 50.0, so the balance is 1050.0.
Input:  future_balance(1000, 0.1)
Output: 1100.0
Explanation: rate overridden to 0.1 for 1 year: interest = 100.0.
Input:  future_balance(1000, 0.1, 3)
Output: 1300.0
Explanation: interest = 1000 * 0.1 * 3 = 300.0, so the balance is 1300.0.

Constraints

  • principal is a number; rate is a fraction; years is a non-negative number.
  • Default rate to 0.05 and years to 1.
  • Use simple interest: principal * rate * years, bound to a local name.

Starter

def future_balance(principal, rate=0.05, years=1):

Because both rate and years have defaults, you can also override just one by name, e.g. future_balance(500, years=2).

6. Discounted Checkout

Define two functions. with_tax(price, tax_rate=0.08) returns a price with sales tax added. Then checkout_total(price, discount=0.0, tax_rate=0.08) applies a discount fraction first, then adds tax by reusing with_tax. A zero discount default means "no discount," so the simplest call just adds tax.

Examples

Input:  with_tax(100)
Output: 108.0
Explanation: 100 + 100 * 0.08 = 108.0 at the default tax rate.
Input:  checkout_total(100)
Output: 108.0
Explanation: discount defaults to 0.0, so this is just with_tax(100).
Input:  checkout_total(100, 0.25)
Output: 81.0
Explanation: 25% off gives 75.0, then 8% tax gives 75 + 6 = 81.0.

Constraints

  • price is a number; discount and tax_rate are fractions.
  • Default discount to 0.0 and tax_rate to 0.08.
  • checkout_total must call with_tax rather than re-deriving the tax (DRY).

Starter

def with_tax(price, tax_rate=0.08):
    pass

def checkout_total(price, discount=0.0, tax_rate=0.08):
    pass
Hint

Discount the price into a local name, then hand that and tax_rate to with_tax.

9. Monthly Loan Payment

Define monthly_payment(principal, annual_rate=0.06, months=12) that returns the fixed monthly payment that fully pays off (amortizes) a loan. The default loan is a one-year loan at 6% annual interest. The amortization formula is

payment = P * r / (1 - (1 + r) ** -months)

where P is the principal and r is the monthly rate. Bind r and the growth factor to local names so the formula reads cleanly, and comment the rate line.

Input:  monthly_payment(1200)

Explanation: r = 0.06/12 = 0.005 over 12 months on a 1200 loan.
Input:  monthly_payment(1200, 0.06, 24)
Output: 53.18473230330935
Explanation: same loan stretched to 24 months halves each payment, roughly.
Input:  monthly_payment(10000, 0.12, 12)
Output: 888.4878867834168
Explanation: r = 0.12/12 = 0.01 over 12 months on a 10000 loan.

Constraints

  • principal is a positive number; annual_rate is a positive fraction; months is a positive integer.
  • Default annual_rate to 0.06 and months to 12.
  • Convert the annual rate to a monthly rate (annual_rate / 12) in the body.
  • Use pow(1 + r, months) for the growth factor; ** is not part of this section's toolbox.

def monthly_payment(principal, annual_rate=0.06, months=12):

Compute r and growth = pow(1 + r, months) first, then the payment is principal * r * growth / (growth - 1).

10. Quadratic Root

Define quadratic_root(a, b, c, sign=1) that returns one real root of a*x**2 + b*x + c using the quadratic formula. The sign parameter selects which root: +1 (the default) takes the larger (-b + sqrt(...))/(2a) branch, and -1 takes the smaller branch. One function, configured by a default, covers both roots.

Examples

Input:  quadratic_root(1, -5, 6)
Output: 3.0
Explanation: x^2 - 5x + 6 = (x-2)(x-3); the larger root (sign +1) is 3.0.
Input:  quadratic_root(1, -5, 6, -1)
Output: 2.0
Explanation: passing sign -1 selects the smaller root, 2.0.
Input:  quadratic_root(2, -4, 2)
Output: 1.0
Explanation: 2x^2 - 4x + 2 = 2(x-1)^2 has a double root at 1.0.

Constraints

  • a is nonzero; b and c are numbers; the discriminant b*b - 4*a*c is non-negative (real roots).
  • Default sign to 1.
  • Bind the discriminant to a local name; use sqrt (shown in Setup).

Starter

def quadratic_root(a, b, c, sign=1):
    pass
Hint

Multiplying the sqrt(discriminant) term by sign (which is +1 or -1) flips between the two roots without any if.

Solutions

Setup

These problems import the operator and math tools from earlier sections and use one float-comparison helper for the irrational answers. Run this block first in a fresh interpreter so every snippet below works:

from operator import add, sub, mul, truediv
from math import sqrt, pi


def almost_equal(a, b, tol=1e-9):
    """Return True when a and b agree to within tol. Test helper only."""
    return abs(a - b) < tol

1. Shipping Cost

Approach — a default that captures the common case. Section 1.4's headline tool is the default argument value: writing rate=2.5 in the header makes rate optional, so the one-argument call uses the standard rate while a caller can still override it. The docstring states the job and documents both parameters.

def shipping_cost(weight, rate=2.5):
    """Return the cost to ship a parcel by weight.

    weight -- parcel weight in kilograms, a number
    rate   -- price charged per kilogram (default: 2.5, the standard rate)
    """
    return weight * rate

Why it works. When rate is omitted, Python binds the header default 2.5 to it; when supplied, the argument wins. The body is one honest multiplication either way.

Complexity. Time O(1), Space O(1). One multiplication, one local frame.

assert shipping_cost(4) == 10.0
assert shipping_cost(10) == 25.0
assert shipping_cost(4, 3.0) == 12.0
assert shipping_cost(0) == 0.0

2. Final Grade

Approach — a default weight plus a derived local name. exam_weight=0.6 is the optional parameter; the homework weight is not a separate input because it is fully determined by the exam weight, so it becomes a local name computed in the body. A comment records the invariant that the two weights sum to 1.

def final_grade(homework, exam, exam_weight=0.6):
    """Return a course grade as a weighted blend of two scores.

    homework    -- the homework score, a number out of 100
    exam        -- the exam score, a number out of 100
    exam_weight -- the exam's share of the grade (default: 0.6)
    """
    homework_weight = 1 - exam_weight  # the two weights always sum to 1
    return homework * homework_weight + exam * exam_weight

Why it works. With the default, the exam counts 0.6 and homework 0.4; overriding exam_weight shifts the blend while keeping the weights summing to 1. final_grade(80, 90, 0.5) makes them equal, giving the plain average 85.0.

Complexity. Time O(1), Space O(1). One extra local binding and a fixed number of operations.

assert final_grade(100, 50) == 70.0
assert final_grade(80, 90) == 86.0
assert final_grade(80, 90, 0.5) == 85.0
assert final_grade(0, 100) == 60.0

3. Circle Area

Approach — fixed constant in the body, variable option as a default. The lesson's design rule: values a caller might reasonably vary become defaulted arguments, while truly fixed values stay in the body. pi never changes, so it sits in the formula; scale might change, so it is an optional parameter defaulting to 1.

def circle_area(radius, scale=1):
    """Return the area of a circle, optionally scaled.

    radius -- the circle's radius, a number
    scale  -- a multiplier applied to the area (default: 1, no scaling)
    """
    return pi * radius * radius * scale

Why it works. pi * radius * radius is the area; multiplying by scale (default 1, so harmless when omitted) lets a caller scale it. The answer is a float because pi is.

Complexity. Time O(1), Space O(1). A constant chain of multiplications.

assert almost_equal(circle_area(1), pi)
assert almost_equal(circle_area(2), 4 * pi)
assert almost_equal(circle_area(1, 2), 2 * pi)
assert almost_equal(circle_area(0), 0.0)

4. Rounded Quotient

Approach — pass a default straight into a built-in. round already accepts a decimal-places argument; the design move is to expose that as an optional parameter places=2, so the common two-place case needs no extra argument and a caller can still ask for more precision.

def rounded_quotient(numerator, denominator, places=2):
    """Return numerator / denominator rounded to a number of places.

    numerator   -- the dividend, a number
    denominator -- the divisor, a nonzero number
    places      -- decimal places to round to (default: 2)
    """
    return round(numerator / denominator, places)

Why it works. The division produces a float, and round(value, places) trims it to the requested precision. Omitting places uses 2; supplying 4 overrides it.

Complexity. Time O(1), Space O(1). One division and one round.

assert rounded_quotient(10, 3) == 3.33
assert rounded_quotient(10, 3, 4) == 3.3333
assert rounded_quotient(1, 8, 3) == 0.125
assert rounded_quotient(7, 2) == 3.5

5. Future Balance

Approach — two defaults plus a named intermediate. Both rate and years are values a caller might vary, so each earns a default (0.05 and 1). The interest earned is a meaningful intermediate, so it is bound to a local name with an explanatory comment before being added back to the principal.

def future_balance(principal, rate=0.05, years=1):
    """Return the balance after applying simple interest.

    principal -- the starting amount, a number
    rate      -- the annual interest rate as a fraction (default: 0.05)
    years     -- the number of years the money is held (default: 1)
    """
    interest = principal * rate * years  # total simple interest earned
    return principal + interest

Why it works. Simple interest is principal * rate * years; adding it to the principal gives the balance. Because both extra parameters have defaults, a caller can override just one by name, as in future_balance(500, years=2).

Complexity. Time O(1), Space O(1). One local binding and a fixed number of operations.

assert future_balance(1000) == 1050.0
assert future_balance(1000, 0.1) == 1100.0
assert future_balance(1000, 0.1, 3) == 1300.0
assert future_balance(500, years=2) == 550.0

6. Discounted Checkout

Approach — compose a helper, don't repeat the tax math (DRY). with_tax owns the one job of adding tax. checkout_total reuses it instead of re-deriving tax, applying the discount first into a local name. A discount default of 0.0 makes "no discount" the easy default call.

def with_tax(price, tax_rate=0.08):
    """Return a price with sales tax added.

    price    -- the pre-tax price, a number
    tax_rate -- the sales-tax fraction (default: 0.08)
    """
    return price + price * tax_rate

def checkout_total(price, discount=0.0, tax_rate=0.08):
    """Return the final price after a discount, then tax.

    price     -- the list price, a number
    discount  -- fraction taken off before tax (default: 0.0, no discount)
    tax_rate  -- the sales-tax fraction passed to with_tax (default: 0.08)
    """
    discounted = price - price * discount  # apply the discount first
    return with_tax(discounted, tax_rate)

Why it works. Discounting 100 by 0.25 gives 75.0; handing that to with_tax at 8% yields 81.0. Because checkout_total calls with_tax, the tax rule is written once and reused — the DRY principle from the lesson.

Complexity. Time O(1), Space O(1). A constant chain of calls; with_tax opens one nested frame.

assert with_tax(100) == 108.0
assert checkout_total(100) == 108.0
assert checkout_total(100, 0.25) == 81.0
assert checkout_total(100, 0.25, 0.0) == 75.0

7. Rectangle Diagonal

Approach — reuse sqrt, default the second side. The diagonal is the square root of the sum of the squared sides. height defaults to 1, so a one-argument call still has a well-defined contract (the diagonal of a unit-tall strip). Squares are written as plain multiplication, since ** is not yet in the toolbox.

def diagonal(width, height=1):
    """Return the diagonal length of a rectangle.

    width  -- the rectangle's width, a number
    height -- the rectangle's height (default: 1, a unit-tall strip)
    """
    return sqrt(width * width + height * height)

Why it works. width * width + height * height is the squared diagonal (Pythagoras); sqrt finishes it. For diagonal(1), height defaults to 1, giving sqrt(2).

Complexity. Time O(1), Space O(1). Two multiplications, one addition, one root.

assert almost_equal(diagonal(3, 4), 5.0)
assert almost_equal(diagonal(1), sqrt(2))
assert almost_equal(diagonal(5, 12), 13.0)
assert almost_equal(diagonal(0), 1.0)

8. Clamp With Defaults

Approach — defaults make a common range free, min/max replace if. No conditionals are available yet, but max and min already encode "at least" and "at most." Defaulting the bounds to 0 and 100 turns the function into a ready-made percentage clamp, while a caller can still pass custom bounds.

def clamp(value, low=0, high=100):
    """Return value pushed into the closed range [low, high].

    value -- the number to constrain
    low   -- the lower bound (default: 0)
    high  -- the upper bound (default: 100)
    """
    return min(max(value, low), high)

Why it works. max(value, low) is never below low; wrapping it in min(..., high) makes the result never above high; an in-range value passes through untouched. Defaults supply the 0..100 bounds when omitted.

Complexity. Time O(1), Space O(1). Two built-in comparisons.

assert clamp(50) == 50
assert clamp(-20) == 0
assert clamp(150) == 100
assert clamp(7, 1, 5) == 5
assert clamp(3, 1, 5) == 3

9. Monthly Loan Payment

Approach — defaults for the common loan, local names for a clear formula. The amortization formula is dense, so naming the monthly rate r and the growth factor turns it into a readable, single-job function. The common one-year 6% loan becomes the default, and pow stands in for **.

def monthly_payment(principal, annual_rate=0.06, months=12):
    """Return the fixed monthly payment that amortizes a loan.

    principal   -- the amount borrowed, a number
    annual_rate -- the yearly interest rate as a fraction (default: 0.06)
    months      -- the number of monthly payments (default: 12)

    Uses the standard amortization formula
        P * r / (1 - (1 + r)^-months)
    where r is the monthly rate.
    """
    r = annual_rate / 12  # the per-month interest rate
    growth = pow(1 + r, months)  # (1 + r) raised to the number of months
    return principal * r * growth / (growth - 1)

Why it works. Multiplying the formula's numerator and denominator by (1 + r) ** months rewrites P*r/(1 - (1+r)**-months) as P*r*growth/(growth - 1), which avoids a negative exponent. The defaults make monthly_payment(1200) a complete call.

Complexity. Time O(1), Space O(1). pow and a fixed number of arithmetic operations.

assert almost_equal(monthly_payment(1200), 103.27971564849884)
assert almost_equal(monthly_payment(1200, 0.06, 24), 53.18473230330935)
assert almost_equal(monthly_payment(10000, 0.12, 12), 888.4878867834168)

10. Quadratic Root

Approach — one function, configured by a default sign. Rather than writing two near-identical functions for the two roots (which would violate DRY), a single sign parameter defaulting to +1 selects the branch. Multiplying the sqrt term by sign flips between roots with no conditional.

def quadratic_root(a, b, c, sign=1):
    """Return one real root of a*x^2 + b*x + c using the quadratic formula.

    a    -- the quadratic coefficient, a nonzero number
    b    -- the linear coefficient, a number
    c    -- the constant term, a number
    sign -- which root to take: +1 for the larger, -1 for the smaller
            (default: 1, the (-b + sqrt(...)) / (2a) branch)
    """
    discriminant = b * b - 4 * a * c  # must be non-negative for real roots
    return (-b + sign * sqrt(discriminant)) / (2 * a)

Why it works. The roots are (-b ± sqrt(b² - 4ac)) / (2a). With sign = 1 the + branch (the larger root) is taken; with sign = -1 the - branch (the smaller root) is taken. For a double root the discriminant is 0, so both branches coincide.

Complexity. Time O(1), Space O(1). One local binding plus a constant chain of operations and one sqrt.

assert almost_equal(quadratic_root(1, -5, 6), 3.0)
assert almost_equal(quadratic_root(1, -5, 6, -1), 2.0)
assert almost_equal(quadratic_root(1, -3, 2), 2.0)
assert almost_equal(quadratic_root(1, -3, 2, -1), 1.0)
assert almost_equal(quadratic_root(2, -4, 2), 1.0)
