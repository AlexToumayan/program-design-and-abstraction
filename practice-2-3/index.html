2.3 · Sequences — Coding Practice

◷ approx. 9 min read · 31 worked examples

These problems build fluency with sequences: indexing and slicing, walking a sequence with a for loop, list comprehensions for transform-and-filter, two-pointer and counting patterns over lists and strings, higher-order tools like filter and reduce, and recursive walks over trees stored as nested lists.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Easy

1. Running Totals

Given a list of numbers prices, return a new list of the same length where the element at each position is the sum of all elements up to and including that position. The original list is left unchanged.

Examples

Input:  running_totals([2, 3, 5])
Output: [2, 5, 10]
Explanation: 2, then 2+3, then 2+3+5.

Input:  running_totals([4, -1, 0, 6])
Output: [4, 3, 3, 9]
Explanation: a negative price lowers the running sum.

Input:  running_totals([])
Output: []
Explanation: an empty input yields an empty list of totals.

Constraints

  • prices is a list of integers (possibly empty).
  • Build the result by walking the list with a for loop; do not mutate any list in place.

Hint

Keep one running number for the sum so far, and concatenate a one-element list onto the result each pass.

Solutions

Setup

These problems use:

from functools import reduce
from operator import mul

Problem 9 also relies on a small tree abstraction (a constructor and selectors), shown with its solution.

1. Running Totals

Approach — running aggregate. Carry one number, so_far, that accumulates the sum as the for loop visits each price, and grow the result list by concatenating a fresh one-element list each pass — the manual version of the transform the section builds.

def running_totals(prices):
    totals = []
    so_far = 0
    for price in prices:
        so_far = so_far + price
        totals = totals + [so_far]
    return totals

Why it works. After visiting position i, so_far equals the sum of prices[0..i], which is exactly the value that belongs at position i of the output.

Complexity. Time O(n2), Space O(n). Each totals + [so_far] copies the growing list, so the copies sum to a quadratic total.

assert running_totals([2, 3, 5]) == [2, 5, 10]
assert running_totals([4, -1, 0, 6]) == [4, 3, 3, 9]

2. Even-Index Sum

Approach — stepped range. range(0, len(s), 2) is a sequence of just the even indices; selecting s[i] at each one and feeding the lot to sum aggregates exactly those positions.

def even_index_sum(s):
    return sum(s[i] for i in range(0, len(s), 2))

Why it works. The third argument to range is the step, so the indices generated are 0, 2, 4, ... up to but not past len(s) - 1, which are the even positions.

Complexity. Time O(n), Space O(1). One pass over half the indices, summing as it goes.

assert even_index_sum([10, 1, 20, 1, 30]) == 60
assert even_index_sum([1, 2, 3, 4]) == 4

3. Initials

Approach — transform then join. Split the name into words, transform each word into its first character with a comprehension, then aggregate the characters into one string with ''.join.

def initials(name):
    words = name.split()
    letters = [word[0] for word in words]
    return ''.join(letters)

Why it works. Indexing a string at 0 returns its first character as a one-character string, and ''.join concatenates a list of strings with no separator between them.

Complexity. Time O(n), Space O(k) for k words. One pass to split and one comprehension over the words.

assert initials('grace brewster hopper') == 'gbh'
assert initials('john von neumann') == 'jvn'

4. Count Vowels

Approach — filter then count. A string is a sequence of characters, so a filtering comprehension keeps only the vowels; the length of that list is the count.

def count_vowels(text):
    return len([c for c in text if c in 'aeiou'])

Why it works. c in 'aeiou' tests whether the single character c appears in the vowel string, so the comprehension keeps exactly the vowel characters, and len counts them.

Complexity. Time O(n), Space O(n). One pass building the filtered list.

assert count_vowels('berkeley') == 3
assert count_vowels('sequence') == 4

5. Two-Sum Exists

Approach — pair scan with index ranges. Two nested range loops generate every pair of distinct positions exactly once by starting the inner index at i + 1, and the first pair that hits the target short-circuits with True.

def two_sum_exists(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return True
    return False

Why it works. Because j always exceeds i, the loops cover each unordered pair once and never pair an element with itself; returning False after both loops means no pair summed to the target.

Complexity. Time O(n2), Space O(1). Every pair of indices may be examined.

assert two_sum_exists([2, 7, 11, 15], 9) is True
assert two_sum_exists([1, 2, 3], 7) is False

6. Palindrome Check

Approach — two-pointer comparison. Drop the spaces with a filter, then compare position i against the mirror position n - 1 - i for the first half; any mismatch decides it.

def is_palindrome(s):
    kept = [c for c in s if c != ' ']
    n = len(kept)
    for i in range(n // 2):
        if kept[i] != kept[n - 1 - i]:
            return False
    return True

Why it works. A sequence is a palindrome exactly when every element equals its mirror; checking the first half against the back half covers all pairs, and the middle element of an odd-length sequence never needs checking.

Complexity. Time O(n), Space O(n). One pass to filter spaces, then half a pass to compare.

assert is_palindrome('never odd or even') is True
assert is_palindrome('hello') is False

7. Flatten One Level

Approach — nested traversal. Walk the outer list, and for each row walk its elements, concatenating each one onto the growing result. This is the closure property of lists turned into a flattening loop.

def flatten_one(rows):
    result = []
    for row in rows:
        for item in row:
            result = result + [item]
    return result

Why it works. The inner loop visits every element of every row in order, and concatenation appends each to result, so the rows end up laid end to end with empty rows contributing nothing.

Complexity. Time O(n2), Space O(n) for n total elements. Each concatenation copies the result built so far.

assert flatten_one([[1, 2], [3], [4, 5, 6]]) == [1, 2, 3, 4, 5, 6]
assert flatten_one([[1, 2], [], [3]]) == [1, 2, 3]

8. Product of Odds

Approach — filter then reduce. This is the assembly line from the section: a filter station keeps the odds, and a reduce station over mul collapses them into one product, starting from 1 so an empty result is handled cleanly.

def product_of_odds(nums):
    odds = filter(lambda x: x % 2 == 1, nums)
    return reduce(mul, odds, 1)

Why it works. filter keeps the elements for which the lambda predicate is true, and reduce(mul, odds, 1) multiplies them onto a running result that begins at 1, the identity for multiplication, so no odds gives 1.

Complexity. Time O(n), Space O(1). One pass to filter and one to multiply, with a constant running value.

assert product_of_odds([1, 2, 3, 4, 5]) == 15
assert product_of_odds([2, 4, 6]) == 1
assert product_of_odds([3, 3, 2]) == 9

9. Tree Depth

Approach — recursion that mirrors the tree. Following the section's pattern, the base case is a leaf, and otherwise the answer is 1 plus the deepest branch, found by recursing on every branch and taking the max.

Why it works. Each non-leaf adds one level above its branches, so max over the branch depths picks the longest path beneath this node, and the leaf base case stops the recursion at the bottom.

Complexity. Time O(n), Space O(h) for n nodes and height h. Every node is visited once; the call stack is as deep as the tree.

assert tree_depth(tree(3, [tree(1), tree(2, [tree(1), tree(1)])])) == 3
assert tree_depth(tree(1, [tree(2, [tree(3, [tree(4)])])])) == 4
assert tree_depth(tree(7)) == 1

10. Run-Length Encode

Approach — single-pass run tracking. Seed the first run from s[0], then scan the rest: extend the current run when the value repeats, and close it off into a [value, count] pair whenever the value changes. A final push records the last run.

def run_length_encode(s):
    if len(s) == 0:
        return []
    runs = []
    current = s[0]
    count = 1
    for x in s[1:]:
        if x == current:
            count = count + 1
        else:
            runs = runs + [[current, count]]
            current = x
            count = 1
    runs = runs + [[current, count]]
    return runs

Why it works. A run ends exactly when a different value appears, so closing the run on each change and pushing the final run after the loop produces one pair per maximal block of equal values, in order.

Complexity. Time O(n2) worst case, Space O(n). The single scan is linear, but each runs + [...] copies the growing list of pairs.

assert run_length_encode([1, 1, 1, 2, 3, 3]) == [[1, 3], [2, 1], [3, 2]]
assert run_length_encode(['a', 'a', 'b', 'a']) == [['a', 2], ['b', 1], ['a', 1]]
assert run_length_encode([]) == []
