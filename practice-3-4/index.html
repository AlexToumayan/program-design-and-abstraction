3.4 · The Calculator Interpreter — Coding Practice

◷ approx. 13 min read · 31 worked examples

These problems build fluency with the Calculator interpreter you assembled in Python: constructing and reading Pair-based expression trees, walking those trees recursively, parsing one line of text with scheme_read, evaluating with calc_eval and calc_apply, and extending or transforming the interpreter itself.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Every problem runs against the Calculator machinery reproduced verbatim in the Setup block at the top of the Solutions section: Pair/nil, the tokenizer, the Buffer and reader (scheme_read/read_tail, plus a one-line read_line helper), and the evaluator (calc_eval/calc_apply with simplify, reduce, and as_scheme_list). Read that block once before you start — it is the same code from this section's teaching documents.

Easy

1. Build a Flat Call Expression

Write make_call(operator, operands) that builds a Calculator call expression as a Pair tree. operator is a symbol string such as '+', and operands is an ordinary Python list of already-built operand values (numbers or subtrees). The result is a Pair whose first is the operator and whose second is the operand list, terminated by nil. Do not call the reader; construct the Pair chain yourself.

Examples

Input:  str(make_call('+', [2, 3]))
Output: (+ 2 3)
Explanation: the operator goes in first; 2 and 3 chain through second to nil.

Input:  repr(make_call('+', [2, 3]))
Output: Pair('+', Pair(2, Pair(3, nil)))
Explanation: repr shows the Python construction; str shows the Scheme list.

Input:  str(make_call('-', [10]))
Output: (- 10)
Explanation: a single operand still forms a well-formed one-element operand list.

Constraints

  • Build the operand list by chaining Pair objects ending in nil; do not use as_scheme_list or the reader.
  • operands may be empty, in which case second is just nil.

Solutions

Setup

These problems run against the Calculator interpreter machinery from this section's teaching documents — Pair/nil, the tokenizer, the Buffer and reader, and the evaluator — plus a one-line read_line helper that parses a single line of text. Define all of it first:

import sys
from operator import add, sub, mul, truediv

class Pair(object):
    """A pair has two instance attributes: first and second."""
    def __init__(self, first, second):
        self.first = first
        self.second = second

    def __repr__(self):
        return "Pair({0}, {1})".format(repr(self.first), repr(self.second))

    def __str__(self):
        s = "(" + str(self.first)
        second = self.second
        while isinstance(second, Pair):
            s += " " + str(second.first)
            second = second.second
        if second is not nil:
            s += " . " + str(second)
        return s + ")"

    def __len__(self):
        n, second = 1, self.second
        while isinstance(second, Pair):
            n += 1
            second = second.second
        if second is not nil:
            raise TypeError("length attempted on improper list")
        return n

    def __getitem__(self, k):
        if k < 0:
            raise IndexError("negative index into list")
        y = self
        for _ in range(k):
            if y.second is nil:
                raise IndexError("list index out of bounds")
            elif not isinstance(y.second, Pair):
                raise TypeError("ill-formed list")
            y = y.second
        return y.first

    def map(self, fn):
        """Return a Scheme list after mapping Python function FN to SELF."""
        mapped = fn(self.first)
        if self.second is nil or isinstance(self.second, Pair):
            return Pair(mapped, self.second.map(fn))
        else:
            raise TypeError("ill-formed list")

class nil(object):
    """The empty list."""
    def __repr__(self):
        return "nil"

    def __str__(self):
        return "()"

    def __len__(self):
        return 0

    def __getitem__(self, k):
        if k < 0:
            raise IndexError("negative index into list")
        raise IndexError("list index out of bounds")

    def map(self, fn):
        return self

nil = nil()  # there is only one empty-list object

_NUMERAL_STARTS = set('0123456789') | set('+-.')
_SYMBOL_CHARS = (set('!$%&*/:<=>?@^_~') | set('abcdefghijklmnopqrstuvwxyz') |
                 set('ABCDEFGHIJKLMNOPQRSTUVWXYZ') | _NUMERAL_STARTS)
_WHITESPACE = set(' \t\n\r')
_SINGLE_CHAR_TOKENS = set("()'`")
_TOKEN_END = _WHITESPACE | _SINGLE_CHAR_TOKENS | {'"', ';'}
DELIMITERS = _SINGLE_CHAR_TOKENS | {'.', ','}

def valid_symbol(s):
    if len(s) == 0:
        return False
    for c in s:
        if c not in _SYMBOL_CHARS:
            return False
    return True

def next_candidate_token(line, k):
    while k < len(line):
        c = line[k]
        if c == ';':
            return None, len(line)
        elif c in _WHITESPACE:
            k += 1
        elif c in _SINGLE_CHAR_TOKENS:
            return c, k + 1
        else:
            j = k
            while j < len(line) and line[j] not in _TOKEN_END:
                j += 1
            return line[k:j], min(j, len(line))
    return None, len(line)

def tokenize_line(line):
    """The list of Scheme tokens on line. Excludes comments and whitespace."""
    result = []
    text, i = next_candidate_token(line, 0)
    while text is not None:
        if text in DELIMITERS:
            result.append(text)
        elif text == '+' or text == '-':
            result.append(text)
        elif text == 'nil':
            result.append(text)
        elif text[0] in _NUMERAL_STARTS:
            try:
                result.append(int(text))
            except ValueError:
                try:
                    result.append(float(text))
                except ValueError:
                    raise ValueError("invalid numeral: {0}".format(text))
        elif text[0] in _SYMBOL_CHARS and valid_symbol(text):
            result.append(text)
        else:
            raise ValueError("invalid token: {0}".format(text))
        text, i = next_candidate_token(line, i)
    return result

def tokenize_lines(input):
    """An iterator returning lists of tokens, one for each line of input."""
    return map(tokenize_line, input)

class Buffer(object):
    """A Buffer provides access to a sequence of tokens across lines."""
    def __init__(self, source):
        self.index = 0
        self.lines = []
        self.source = source
        self.current_line = ()
        self.current()

    def pop(self):
        """Remove the next item from self and return it, or None if exhausted."""
        current = self.current()
        self.index += 1
        return current

    @property
    def more_on_line(self):
        return self.index < len(self.current_line)

    def current(self):
        """Return the current element, or None if none exists."""
        while not self.more_on_line:
            self.index = 0
            try:
                self.current_line = next(self.source)
                self.lines.append(self.current_line)
            except StopIteration:
                self.current_line = ()
                return None
        return self.current_line[self.index]

def scheme_read(src):
    """Read the next expression from src, a Buffer of tokens."""
    if src.current() is None:
        raise EOFError
    val = src.pop()
    if val == 'nil':
        return nil
    elif val not in DELIMITERS:
        return val
    elif val == "(":
        return read_tail(src)
    else:
        raise SyntaxError("unexpected token: {0}".format(val))

def read_tail(src):
    """Return the remainder of a list in src, starting before an element or )."""
    if src.current() is None:
        raise SyntaxError("unexpected end of file")
    if src.current() == ")":
        src.pop()
        return nil
    first = scheme_read(src)
    rest = read_tail(src)
    return Pair(first, rest)

def read_line(line):
    """Read a single Calculator expression from one line of text."""
    return scheme_read(Buffer(tokenize_lines([line])))

def simplify(value):
    """Return an int if value is a whole float, else value unchanged."""
    if isinstance(value, float) and int(value) == value:
        return int(value)
    return value

def reduce(fn, scheme_list, start):
    """Reduce a recursive list of Pairs using fn and a start value."""
    if scheme_list is nil:
        return start
    return reduce(fn, scheme_list.second, fn(start, scheme_list.first))

def as_scheme_list(*args):
    """Return a recursive list of Pairs containing the elements of args."""
    if len(args) == 0:
        return nil
    return Pair(args[0], as_scheme_list(*args[1:]))

def calc_apply(operator, args):
    """Apply the named operator to a Scheme list of args."""
    if not isinstance(operator, str):
        raise TypeError(str(operator) + ' is not a symbol')
    if operator == '+':
        return reduce(add, args, 0)
    elif operator == '-':
        if len(args) == 0:
            raise TypeError(operator + ' requires at least 1 argument')
        elif len(args) == 1:
            return -args.first
        else:
            return reduce(sub, args.second, args.first)
    elif operator == '*':
        return reduce(mul, args, 1)
    elif operator == '/':
        if len(args) == 0:
            raise TypeError(operator + ' requires at least 1 argument')
        elif len(args) == 1:
            return 1 / args.first
        else:
            return reduce(truediv, args.second, args.first)
    else:
        raise TypeError(operator + ' is an unknown operator')

def calc_eval(exp):
    """Evaluate a Calculator expression."""
    if type(exp) in (int, float):
        return simplify(exp)
    elif isinstance(exp, Pair):
        arguments = exp.second.map(calc_eval)
        return simplify(calc_apply(exp.first, arguments))
    else:
        raise TypeError(str(exp) + ' is not a number or call expression')

1. Build a Flat Call Expression

Approach — chain Pairs from the back. A Scheme list is a chain of Pair boxes ending in nil, so build the operand list by starting from nil and wrapping each operand (in reverse order) in a new Pair. Then put the operator in first — the same (operator operand-1 ... operand-n) shape the reader produces.

def make_call(operator, operands):
    result = nil
    for value in reversed(operands):
        result = Pair(value, result)
    return Pair(operator, result)

Why it works. Wrapping operands in reverse means the last operand attaches to nil first and the first operand ends up at the front of the chain, exactly matching Pair(op, Pair(o1, Pair(o2, nil))).

Complexity. Time O(n), Space O(n) for n operands. One Pair is built per operand.

assert str(make_call('+', [2, 3])) == '(+ 2 3)'
assert str(make_call('*', [4, 5, 6])) == '(* 4 5 6)'
assert str(make_call('-', [10])) == '(- 10)'
assert repr(make_call('+', [2, 3])) == "Pair('+', Pair(2, Pair(3, nil)))"
assert make_call('+', [2, 3]).first == '+'
assert len(make_call('+', [2, 3]).second) == 2

2. Read Operator and Arity

Approach — first and second. Every combination stores the operator in first and the operand list in second. Because a Pair chain (and nil) supports len, one length call returns the arity directly, including the zero-operand case.

def operator_and_arity(exp):
    operator = exp.first
    operands = exp.second
    return (operator, len(operands))

Why it works. exp.second is the well-formed operand list; len walks it to nil counting elements, and len(nil) is 0, so the function handles (+) and (/ 40 (* 4 5)) alike.

Complexity. Time O(n), Space O(1) for n operands. len walks the operand chain once.

assert operator_and_arity(read_line('(+ 2 3)')) == ('+', 2)
assert operator_and_arity(read_line('(* 4 5 6)')) == ('*', 3)
assert operator_and_arity(read_line('(- 9)')) == ('-', 1)
assert operator_and_arity(read_line('(/ 40 (* 4 5))')) == ('/', 2)
assert operator_and_arity(read_line('(+)')) == ('+', 0)

3. Evaluate One Line

Approach — Read then Eval. This is the heart of the REPL for a single expression: parse the text into a Pair tree with read_line, then hand that tree to calc_eval, which dispatches on type and recurses through the nesting.

def calc(line):
    return calc_eval(read_line(line))

Why it works. read_line turns the characters into the expression tree, and calc_eval evaluates operands before applying the operator, so any depth of nesting resolves inside-out to a single value.

Complexity. Time O(n), Space O(d) where n is the token count and d the nesting depth. Parsing and evaluation each visit every node once; recursion depth is the tree depth.

assert calc('(+ 2 3)') == 5
assert calc('(* (+ 1 2) (+ 2 3))') == 15
assert calc('(- 100 (* 7 (+ 8 (/ -12 -3))))') == 16
assert calc('(+)') == 0
assert calc('(/ 10)') == 0.1
assert calc('42') == 42

4. Count the Numbers in a Tree

Approach — tree recursion over operands. A leaf (non-Pair) contributes one number; a combination contributes the sum over its operands. Walking exp.second with a while loop and recursing into each operands.first visits exactly the operands, never the operator symbol in first.

def count_numbers(exp):
    if isinstance(exp, Pair):
        total = 0
        operands = exp.second
        while operands is not nil:
            total = total + count_numbers(operands.first)
            operands = operands.second
        return total
    else:
        return 1

Why it works. The recursion mirrors calc_eval's structure but counts instead of evaluates: it descends only into operands, so operator symbols never reach the leaf case.

Complexity. Time O(n), Space O(d) for a tree of n nodes and depth d. Each node is visited once; the call stack is the tree depth.

assert count_numbers(read_line('(+ 2 3)')) == 2
assert count_numbers(read_line('(* (+ 1 2) (+ 2 3))')) == 4
assert count_numbers(read_line('(- 100 (* 7 (+ 8 (/ -12 -3))))')) == 5
assert count_numbers(read_line('(+)')) == 0
assert count_numbers(read_line('7')) == 1

5. Sum Every Number

Approach — same walk, numeric leaf. Reuse the operand-recursion shape, but the leaf case returns the number itself rather than 1. Operators are skipped because only exp.second operands are recursed into, so no operator symbol is ever added.

def sum_leaves(exp):
    if isinstance(exp, Pair):
        total = 0
        operands = exp.second
        while operands is not nil:
            total = total + sum_leaves(operands.first)
            operands = operands.second
        return total
    else:
        return exp

Why it works. Every number leaf is reached exactly once and added in; combinations contribute the sum of their operands' sums, so the total is the sum of all leaves regardless of which operators appear.

Complexity. Time O(n), Space O(d) for n nodes and depth d. One visit per node; stack depth is the tree depth.

assert sum_leaves(read_line('(+ 2 3)')) == 5
assert sum_leaves(read_line('(* (+ 1 2) (+ 2 3))')) == 8
assert sum_leaves(read_line('(- 100 (* 7 (+ 8 (/ -12 -3))))')) == 100
assert sum_leaves(read_line('(+ 10 (- 5))')) == 15
assert sum_leaves(read_line('9')) == 9

6. Depth of an Expression Tree

Approach — one plus the deepest operand. A non-Pair has depth 0. A combination is 1 plus the largest depth among its operands, defaulting to 0 when it has none, so a flat (+ 2 3) is depth 1 and each nested layer adds one.

def tree_depth(exp):
    if not isinstance(exp, Pair):
        return 0
    deepest = 0
    operands = exp.second
    while operands is not nil:
        child = tree_depth(operands.first)
        if child > deepest:
            deepest = child
        operands = operands.second
    return 1 + deepest

Why it works. The recursion computes each operand's depth and keeps the maximum; adding 1 accounts for the current combination's own level, which is the standard tree-height definition.

Complexity. Time O(n), Space O(d) for n nodes and depth d. Every node contributes one comparison; recursion depth equals the tree depth.

assert tree_depth(read_line('7')) == 0
assert tree_depth(read_line('(+ 2 3)')) == 1
assert tree_depth(read_line('(* (+ 1 2) (+ 2 3))')) == 2
assert tree_depth(read_line('(- 100 (* 7 (+ 8 (/ -12 -3))))')) == 4
assert tree_depth(read_line('(+ 1 (* 2 (+ 3 (* 4 5))))')) == 4

7. Add a max Operator

Approach — extend apply, mirror eval. Adding an operator to the Calculator means teaching calc_apply a new clause and routing combinations through it. apply_with_max handles 'max' by scanning the already-evaluated args list and otherwise defers to the original calc_apply; eval_with_max is calc_eval with the two calls swapped to point at the extended versions.

def apply_with_max(operator, args):
    if operator == 'max':
        if len(args) == 0:
            raise TypeError('max requires at least 1 argument')
        best = args.first
        rest = args.second
        while rest is not nil:
            if rest.first > best:
                best = rest.first
            rest = rest.second
        return best
    return calc_apply(operator, args)

def eval_with_max(exp):
    if type(exp) in (int, float):
        return simplify(exp)
    elif isinstance(exp, Pair):
        arguments = exp.second.map(eval_with_max)
        return simplify(apply_with_max(exp.first, arguments))
    else:
        raise TypeError(str(exp) + ' is not a number or call expression')

Why it works. Because eval_with_max maps itself over the operands, nested max and arithmetic calls are evaluated to values first, so apply_with_max always receives finished numbers — exactly the contract calc_apply relies on. Unknown-to-max operators fall straight through to the existing arithmetic.

Complexity. Time O(n), Space O(d) for n nodes and depth d. Each combination scans its operands once; recursion depth is the tree depth.

assert eval_with_max(read_line('(max 3 7 2)')) == 7
assert eval_with_max(read_line('(max 5)')) == 5
assert eval_with_max(read_line('(max (+ 1 2) (* 2 2))')) == 4
assert eval_with_max(read_line('(+ 1 (max 4 9 2))')) == 10
assert eval_with_max(read_line('(+ 2 3)')) == 5

8. Reverse Every Operand List

Approach — rebuild reversed Pairs. Keep the operator in first, but build a new operand list that prepends each recursively-reversed operand onto a nil-seeded accumulator. Prepending while iterating front-to-back naturally reverses the order, and recursing into each operand reverses subtrees too.

def reverse_operands(exp):
    if not isinstance(exp, Pair):
        return exp
    new_operands = nil
    operands = exp.second
    while operands is not nil:
        reversed_child = reverse_operands(operands.first)
        new_operands = Pair(reversed_child, new_operands)
        operands = operands.second
    return Pair(exp.first, new_operands)

Why it works. Each operand is processed in order but pushed onto the front of new_operands, so the first operand ends up last; the recursive call reverses each operand subtree before it is placed. No original Pair is modified — every node is freshly constructed.

Complexity. Time O(n), Space O(n) for a tree of n nodes. Every node is rebuilt once.

assert str(reverse_operands(read_line('(- 10 1 2)'))) == '(- 2 1 10)'
assert str(reverse_operands(read_line('(/ 40 (* 4 5))'))) == '(/ (* 5 4) 40)'
assert str(reverse_operands(read_line('(+ 2 3)'))) == '(+ 3 2)'
assert str(reverse_operands(read_line('7'))) == '7'
assert calc_eval(reverse_operands(read_line('(- 10 1 2)'))) == -9

9. Constant-Fold the Tree

Approach — fold leaves up, evaluate all-number nodes. Recurse first so every operand is folded, tracking whether all folded operands turned into numbers. Rebuild the combination from the folded operands; if every one is a number, the whole combination is constant, so evaluate it with calc_eval and return the single value. Otherwise return the rebuilt (partially folded) combination. Number and symbol leaves pass straight through.

def constant_fold(exp):
    if not isinstance(exp, Pair):
        return exp
    folded_operands = []
    all_numbers = True
    operands = exp.second
    while operands is not nil:
        folded = constant_fold(operands.first)
        folded_operands.append(folded)
        if type(folded) not in (int, float):
            all_numbers = False
        operands = operands.second
    new_operands = nil
    for value in reversed(folded_operands):
        new_operands = Pair(value, new_operands)
    new_exp = Pair(exp.first, new_operands)
    if all_numbers:
        return calc_eval(new_exp)
    return new_exp

Why it works. Folding bottom-up means by the time a combination is examined, each operand is either a number or an irreducible subtree. A combination with only numeric operands is exactly what calc_eval can collapse to one value; any non-number operand (a variable symbol) keeps the combination symbolic while its numeric siblings are already folded.

Complexity. Time O(n), Space O(n) for a tree of n nodes. Each node is folded and rebuilt once; the all-number calc_eval calls re-traverse only already-collapsed subtrees.

assert constant_fold(read_line('(+ 2 3)')) == 5
assert constant_fold(read_line('(* (+ 1 2) (+ 2 3))')) == 15
assert str(constant_fold(read_line('(+ x (* 2 3))'))) == '(+ x 6)'
assert str(constant_fold(read_line('(- (+ 1 2) y)'))) == '(- 3 y)'
assert constant_fold(read_line('z')) == 'z'
assert str(constant_fold(read_line('(* w (+ a 1))'))) == '(* w (+ a 1))'

10. Pretty-Print as Infix

Approach — recurse and join. A leaf prints as its own text. A combination collects the infix form of each operand into a Python list; a single operand prints as the operator directly in front of it, while two or more are joined by ' operator ' and wrapped in parentheses. Recursing into operands renders subtrees, and the parentheses preserve grouping.

def to_infix(exp):
    if not isinstance(exp, Pair):
        return str(exp)
    operator = exp.first
    operands = exp.second
    if len(operands) == 1:
        return '(' + operator + to_infix(operands.first) + ')'
    parts = []
    rest = operands
    while rest is not nil:
        parts.append(to_infix(rest.first))
        rest = rest.second
    joined = (' ' + operator + ' ').join(parts)
    return '(' + joined + ')'

Why it works. Each operand is converted before being joined, so nested combinations become parenthesized infix strings in place. Wrapping every multi-operand combination in parentheses keeps the original grouping unambiguous, and the unary case mirrors how (- 3) reads as negation (-3).

Complexity. Time O(n), Space O(n) for a tree of n nodes. Each node produces one string fragment; joining concatenates them once.

assert to_infix(read_line('(+ 2 3)')) == '(2 + 3)'
assert to_infix(read_line('(* (+ 1 2) (+ 2 3))')) == '((1 + 2) * (2 + 3))'
assert to_infix(read_line('(- 10 1 2)')) == '(10 - 1 - 2)'
assert to_infix(read_line('(- 3)')) == '(-3)'
assert to_infix(read_line('(/ 40 (* 4 5))')) == '(40 / (4 * 5))'
assert to_infix(read_line('7')) == '7'
