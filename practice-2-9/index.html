Solutions

Setup

Every solution below operates on the Link and Tree classes from this section, reproduced here verbatim. Paste this block into a fresh interpreter first, then paste any solution beneath it.

1. Sum a Linked List

Approach — recursion on first and rest. A linked list is a first followed by a rest that is itself a linked list, so the sum is s.first plus the sum of s.rest. The empty chain is the base case, contributing 0.

def link_sum(s):
    if s is Link.empty:
        return 0
    else:
        return s.first + link_sum(s.rest)

Why it works. Every value is visited exactly once as the recursion peels off one link per call, and the empty-chain base case stops the walk while contributing nothing.

Complexity. Time O(n), Space O(n). One call per link, and the call stack is as deep as the chain is long.

assert link_sum(Link.empty) == 0
assert link_sum(Link(7)) == 7
assert link_sum(Link(3, Link(4, Link(5)))) == 12
assert link_sum(Link(-2, Link(2, Link(10)))) == 10

2. Count a Value in a Linked List

Approach — recursion that adds on a match. Walk the chain link by link. When s.first equals v, add 1 to the count from the rest; otherwise just take the count from the rest.

def count_link(s, v):
    if s is Link.empty:
        return 0
    elif s.first == v:
        return 1 + count_link(s.rest, v)
    else:
        return count_link(s.rest, v)

Why it works. The two recursive branches differ only by whether the current link adds 1, so every match anywhere in the chain is counted exactly once, and the empty chain contributes 0.

Complexity. Time O(n), Space O(n). Every link is inspected once, with recursion depth equal to the chain length.

assert count_link(Link.empty, 4) == 0
assert count_link(Link(4, Link(2, Link(4, Link(4)))), 4) == 3
assert count_link(Link(4, Link(2, Link(4, Link(4)))), 2) == 1
assert count_link(Link(1, Link(2, Link(3))), 9) == 0

3. Largest Label in a Tree

Approach — recurse into each branch. Because every node carries a label, the largest label is the bigger of this node's label and the largest label found in any branch. A leaf has no branches, so its answer is simply its own label.

def max_label(t):
    largest = t.label
    for b in t.branches:
        branch_max = max_label(b)
        if branch_max > largest:
            largest = branch_max
    return largest

Why it works. The loop folds each branch's recursively computed maximum against largest, so by the time it finishes, largest is the maximum over this node and every descendant. The empty-branch case (a leaf) skips the loop entirely.

Complexity. Time O(n), Space O(h) for n nodes and tree height h. Every node is visited once; the stack depth follows the height.

assert max_label(Tree(5)) == 5
assert max_label(Tree(3, (Tree(1), Tree(2)))) == 3
assert max_label(Tree(3, (Tree(9), Tree(2, (Tree(4), Tree(7)))))) == 9
assert max_label(Tree(-1, (Tree(-5), Tree(-3)))) == -1

4. Count the Leaves of a Tree

Approach — leaves are the base case. A leaf contributes 1. An internal node contributes the total leaf count of its branches, so recurse into each branch and add up the results.

def count_leaves(t):
    if t.is_leaf():
        return 1
    else:
        total = 0
        for b in t.branches:
            total = total + count_leaves(b)
        return total

Why it works. t.is_leaf() is True exactly when t.branches is empty, which both stops the recursion and identifies the things being counted. Internal nodes contribute nothing themselves; they pass the count up from their branches.

Complexity. Time O(n), Space O(h) for n nodes and tree height h. Each node is visited once; recursion depth follows the height.

assert count_leaves(Tree(5)) == 1
assert count_leaves(Tree(3, (Tree(1), Tree(2)))) == 2
assert count_leaves(Tree(3, (Tree(1), Tree(2, (Tree(4), Tree(7)))))) == 3
assert count_leaves(Tree(1, (Tree(2, (Tree(3, (Tree(4),)),)),))) == 1

5. Reverse a Linked List

Approach — accumulate onto the front. Walk s forward, and at each link push s.first onto the front of a reversed-so-far chain. When s runs out, that accumulator is the full reversal.

def reverse_link(s):
    def build(s, reversed_so_far):
        if s is Link.empty:
            return reversed_so_far
        else:
            return build(s.rest, Link(s.first, reversed_so_far))
    return build(s, Link.empty)

Why it works. The first value of s is wrapped first, so it ends up deepest in the accumulator, which is exactly the back of the reversed list. Building new links never touches s, so the original is unchanged.

Complexity. Time O(n), Space O(n). One new link per element, and the recursion is as deep as the chain.

assert reverse_link(Link.empty) is Link.empty
assert repr(reverse_link(Link(1, Link(2, Link(3))))) == 'Link(3, Link(2, Link(1)))'
assert repr(reverse_link(Link(7))) == 'Link(7)'
assert repr(reverse_link(Link(4, Link(4, Link(5))))) == 'Link(5, Link(4, Link(4)))'

6. Double Every Value

Approach — the map pattern. Build a fresh link whose first is s.first * 2 and whose rest is the doubled remainder. The empty chain maps to itself, so the result is a new chain of the same length.

def double_link(s):
    if s is Link.empty:
        return s
    else:
        return Link(s.first * 2, double_link(s.rest))

Why it works. Each link's transformed value is placed in front of the recursively transformed rest, so the new chain has the same shape as s with every value doubled. Returning s in the base case supplies the Link.empty tail.

Complexity. Time O(n), Space O(n). One new link per element, with recursion depth equal to the chain length.

assert double_link(Link.empty) is Link.empty
assert repr(double_link(Link(1, Link(2, Link(3))))) == 'Link(2, Link(4, Link(6)))'
assert repr(double_link(Link(0, Link(-3)))) == 'Link(0, Link(-6))'
assert repr(double_link(Link(5))) == 'Link(10)'

7. Keep Values Above a Threshold

Approach — the filter pattern. Filter the rest first, then decide about the first value: keep it in front of the filtered rest when s.first > cutoff, otherwise drop it and return the filtered rest alone.

def above(s, cutoff):
    if s is Link.empty:
        return s
    else:
        rest = above(s.rest, cutoff)
        if s.first > cutoff:
            return Link(s.first, rest)
        else:
            return rest

Why it works. Computing the filtered rest before deciding about s.first keeps the surviving values in their original order. Dropping a value simply returns rest without wrapping s.first, so it never reaches the output.

Complexity. Time O(n), Space O(n). Every link is examined once; the result holds at most n links and the stack is n deep.

assert above(Link.empty, 3) is Link.empty
assert repr(above(Link(1, Link(5, Link(3, Link(8)))), 3)) == 'Link(5, Link(8))'
assert above(Link(1, Link(2, Link(3))), 5) is Link.empty
assert repr(above(Link(10, Link(20, Link(30))), 0)) == 'Link(10, Link(20, Link(30)))'

8. Height of a Tree

Approach — one plus the tallest branch. A leaf has height 0. For an internal node, find the tallest branch height and add 1 for the edge down from this node to that branch.

def height(t):
    if t.is_leaf():
        return 0
    else:
        tallest = 0
        for b in t.branches:
            branch_height = height(b)
            if branch_height > tallest:
                tallest = branch_height
        return 1 + tallest

Why it works. Each branch reports the longest downward path within it; taking the maximum and adding the single edge to that branch gives the longest path from this node. Leaves stop the recursion at 0.

Complexity. Time O(n), Space O(h) for n nodes and height h. Each node is visited once; the stack depth follows the height.

assert height(Tree(5)) == 0
assert height(Tree(3, (Tree(1), Tree(2)))) == 1
assert height(Tree(3, (Tree(1), Tree(2, (Tree(4), Tree(7)))))) == 2
assert height(Tree(1, (Tree(2, (Tree(3, (Tree(4),)),)),))) == 3

9. Root-to-Leaf Path Sums

Approach — map then concatenate. A leaf's only path sum is its own label, Link(t.label). For an internal node, recurse into each branch, add t.label to every sum that branch returns (the map step, add_to_each), and glue the branch results together in order (join_links, the extend pattern from this section).

def join_links(s, t):
    if s is Link.empty:
        return t
    else:
        return Link(s.first, join_links(s.rest, t))

def leaf_path_sums(t):
    if t.is_leaf():
        return Link(t.label)
    else:
        result = Link.empty
        for b in t.branches:
            branch_sums = leaf_path_sums(b)
            shifted = add_to_each(branch_sums, t.label)
            result = join_links(result, shifted)
        return result

def add_to_each(s, n):
    if s is Link.empty:
        return s
    else:
        return Link(s.first + n, add_to_each(s.rest, n))

Why it works. Each branch returns the path sums measured from the branch root; adding t.label to every one of them extends each path up through the current node. Concatenating branches left to right keeps the leaves in left-to-right order, and a leaf supplies the single-element base case.

Complexity. Time O(n⋅h), Space O(n) for n nodes and height h. Each leaf's sum is shifted once per ancestor as it bubbles up, and the result holds one entry per leaf.

assert repr(leaf_path_sums(Tree(5))) == 'Link(5)'
assert repr(leaf_path_sums(Tree(1, (Tree(2), Tree(3))))) == 'Link(3, Link(4))'
assert repr(
    leaf_path_sums(Tree(1, (Tree(2, (Tree(4), Tree(5))), Tree(3))))
) == 'Link(7, Link(8, Link(4)))'
assert repr(
    leaf_path_sums(Tree(10, (Tree(1, (Tree(100),)), Tree(2))))
) == 'Link(111, Link(12))'

10. Does a Linked List Contain a Sublist?

Approach — prefix match at each position. A helper starts_with checks whether pattern matches the front of s link by link. Then contains_sublist tries that match at the head; if it fails, it slides one link to the right by recursing on s.rest.

def starts_with(s, pattern):
    if pattern is Link.empty:
        return True
    elif s is Link.empty:
        return False
    elif s.first == pattern.first:
        return starts_with(s.rest, pattern.rest)
    else:
        return False

def contains_sublist(s, pattern):
    if starts_with(s, pattern):
        return True
    elif s is Link.empty:
        return False
    else:
        return contains_sublist(s.rest, pattern)

Why it works. starts_with returns True as soon as the whole pattern is consumed and False the moment a value disagrees or s runs out mid-pattern. contains_sublist tries every starting link, so it finds a consecutive run wherever one exists; the empty pattern matches immediately at the first position.

Complexity. Time O(n⋅m), Space O(n+m) for chains of length n and m. Each of n start positions runs a prefix check up to length m.

assert contains_sublist(Link(1, Link(2, Link(3))), Link.empty) is True
assert contains_sublist(Link(1, Link(2, Link(3, Link(4)))), Link(2, Link(3))) is True
assert contains_sublist(Link(1, Link(2, Link(3, Link(4)))), Link(3, Link(2))) is False
assert contains_sublist(Link(5, Link(6, Link(7))), Link(6, Link(7))) is True
assert contains_sublist(Link(5, Link(6, Link(7))), Link(7, Link(8))) is False
assert contains_sublist(Link.empty, Link(1)) is False
