2.4 · Mutable Data — Coding Practice

◷ approx. 11 min read · 30 worked examples

These problems build fluency with mutable data: changing a list in place, telling identity (is) apart from equality (==), counting and grouping with dictionaries, and giving a function memory through local state, nonlocal, and message-passing dispatch.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Easy

1. Restock the Shelf

Examples

Input:  restock(['milk', 'eggs', 'bread'], 'eggs', 'butter')
Output: ['milk', 'bread', 'butter']
Explanation: 'eggs' is removed, 'butter' is appended to the end.

Input:  restock(['a', 'b', 'c', 'b'], 'b', 'd')
Output: ['a', 'c', 'b', 'd']
Explanation: remove only the first 'b', then append 'd'.

Output: ['y']
Explanation: the one item is removed and the new item appended.

Constraints

  • sold is guaranteed to be present in shelf.
  • Mutate the list in place with list methods; do not build a new list.

Solutions

1. Restock the Shelf

Approach — mutate in place with remove and append. This section's first lesson is that lists are mutable: remove deletes the first element equal to its argument, and append adds one element to the end. Both change the existing list object rather than building a new one, so returning shelf returns the very list that was passed in.

def restock(shelf, sold, added):
    shelf.remove(sold)
    shelf.append(added)
    return shelf

Why it works. remove(sold) takes out the first occurrence and append(added) puts the new product on the end, both in place; the name shelf still points at the same list, now with updated contents.

Complexity. Time O(n), Space O(1) — remove scans for the match; no new list is allocated.

s1 = ['milk', 'eggs', 'bread']
assert restock(s1, 'eggs', 'butter') == ['milk', 'bread', 'butter']
assert s1 == ['milk', 'bread', 'butter']
s2 = ['a', 'b', 'c', 'b']
assert restock(s2, 'b', 'd') == ['a', 'c', 'b', 'd']
result = restock(['x'], 'x', 'y')
assert result == ['y']

2. Same List or Just Equal?

Approach — is for identity, == for equality. This section draws the sharp line between the two operators: is asks whether two expressions evaluate to the exact same object, while == asks whether their contents are equal. Test is first because identical objects are always equal; only when they are separate objects does the equality check distinguish "equal but separate" from "different".

def relationship(a, b):
    if a is b:
        return 'same object'
    elif a == b:
        return 'equal but separate'
    else:
        return 'different'

Why it works. Two names bound to one object pass is. Two distinct objects fail is but pass == when their contents match; otherwise they fail both.

Complexity. Time O(n), Space O(1) — == compares contents; is is a constant-time pointer check.

shared = [1, 2, 3]
assert relationship(shared, shared) == 'same object'
assert relationship([1, 2, 3], [1, 2, 3]) == 'equal but separate'
assert relationship([1, 2, 3], [1, 2, 4]) == 'different'
assert relationship([], []) == 'equal but separate'

3. Tally the Votes

Approach — dictionary counting with get. A dictionary maps each distinct vote to its running count. The get method supplies a fallback of 0 the first time a key is seen, so counts[v] = counts.get(v, 0) + 1 works whether or not v is already a key — no KeyError, no special first-time case.

def tally(votes):
    counts = {}
    for v in votes:
        counts[v] = counts.get(v, 0) + 1
    return counts

Why it works. Each vote either starts a new entry at 1 (via the 0 fallback) or bumps an existing entry by one, so after the loop every key holds its exact frequency.

Complexity. Time O(n), Space O(k) for k distinct votes — one dictionary lookup-and-store per vote.

assert tally(['yes', 'no', 'yes', 'yes']) == {'yes': 3, 'no': 1}
assert tally([]) == {}
assert tally(['a']) == {'a': 1}
assert tally([1, 1, 2, 3, 3, 3]) == {1: 2, 2: 1, 3: 3}

4. Squares Table

Approach — dictionary comprehension. A dictionary comprehension runs a loop and produces one key: value entry per pass, collecting them into a new dictionary. Here the key is n and the value is n * n, drawn from range(lo, hi + 1) so that hi is included. An empty range simply yields an empty dictionary.

def squares_table(lo, hi):
    return {n: n * n for n in range(lo, hi + 1)}

Why it works. range(lo, hi + 1) produces exactly lo through hi; the comprehension files each one under itself with its square as the value.

Complexity. Time O(n), Space O(n) for n = hi - lo + 1 entries — one pass over the range.

assert squares_table(1, 4) == {1: 1, 2: 4, 3: 9, 4: 16}
assert squares_table(5, 5) == {5: 25}
assert squares_table(0, 2) == {0: 0, 1: 1, 2: 4}
assert squares_table(3, 2) == {}

5. Running Total

Approach — local state with nonlocal. The outer make_accumulator holds a private total in its frame; the inner accumulate rebinds that total with each call. The nonlocal total declaration is what redirects the assignment outward to the enclosing frame instead of creating a fresh local that vanishes when the call returns. Each call to make_accumulator builds a new frame, so separate accumulators keep separate totals.

def make_accumulator():
    total = 0
    def accumulate(amount):
        nonlocal total
        total = total + amount
        return total
    return accumulate

Why it works. accumulate reads and updates the one total in its parent frame, which survives between calls, so the running sum persists; a second make_accumulator() call creates an independent total.

Complexity. Time O(1) per call, Space O(1) — a single number is carried in the enclosing frame.

acc = make_accumulator()
assert acc(10) == 10
assert acc(5) == 15
assert acc(-3) == 12
acc2 = make_accumulator()
assert acc2(100) == 100
assert acc(1) == 13

6. Group by Length

Approach — dict-based grouping with appended lists. The dictionary maps each length to a list of the words that share it. The first time a length appears, create an empty list under that key; then append the current word. Because the values are mutable lists, appending updates them in place, and the words stay in their original order.

def group_by_length(words):
    groups = {}
    for w in words:
        n = len(w)
        if n not in groups:
            groups[n] = []
        groups[n].append(w)
    return groups

Why it works. The if n not in groups guard guarantees a list exists before the append, so each word lands in its length's bucket in encounter order.

Complexity. Time O(n), Space O(n) — every word is looked up once and appended once.

assert group_by_length(['hi', 'cat', 'go', 'dog']) == {2: ['hi', 'go'], 3: ['cat', 'dog']}
assert group_by_length([]) == {}
assert group_by_length(['a', 'bb', 'cc', 'd']) == {1: ['a', 'd'], 2: ['bb', 'cc']}
assert group_by_length(['same', 'size']) == {4: ['same', 'size']}

7. Merge Inventories

Approach — copy, then sum with items and get. To avoid mutating either argument, start from a fresh copy of a with the dict constructor. Then walk the second inventory's (item, count) pairs via items, and use get with a fallback of 0 so an item missing from the copy starts at zero before its count is added.

def merge_inventories(a, b):
    merged = dict(a)
    for item, count in b.items():
        merged[item] = merged.get(item, 0) + count
    return merged

Why it works. dict(a) builds a new dictionary with a's contents, leaving a untouched; each pair from b either adds to an existing count or starts a new entry, so shared items are summed.

Complexity. Time O(n+m), Space O(n+m) for inventories of size n and m — the copy plus one pass over b.

inv_a = {'apple': 3, 'pear': 2}
assert merge_inventories(inv_a, {'pear': 5, 'plum': 1}) == {'apple': 3, 'pear': 7, 'plum': 1}
assert inv_a == {'apple': 3, 'pear': 2}
assert merge_inventories({}, {'x': 1}) == {'x': 1}
assert merge_inventories({'x': 1}, {}) == {'x': 1}

8. Bounded Counter

Approach — stateful dispatch function. A single inner dispatch function reads its message argument and selects a behavior, the message-passing pattern from this section. The private count lives in the enclosing frame; nonlocal count lets dispatch rebind it. The 'tick' branch increments only while below limit, so the count saturates rather than overflowing.

def make_counter(limit):
    count = 0
    def dispatch(message):
        nonlocal count
        if message == 'tick':
            if count < limit:
                count = count + 1
            return count
        elif message == 'reset':
            count = 0
            return count
        elif message == 'value':
            return count
    return dispatch

Why it works. The hidden count persists between calls in the parent frame; each message routes to its branch, and the count < limit guard caps the value at limit.

Complexity. Time O(1) per message, Space O(1) — one integer of state.

c = make_counter(2)
assert c('value') == 0
assert c('tick') == 1
assert c('tick') == 2
assert c('tick') == 2
assert c('reset') == 0
assert c('tick') == 1

9. Dedupe in Place

Approach — index walk with pop and a seen-dictionary. Walk an index i along the list. A dictionary seen records every value already kept. When seq[i] is a repeat, pop(i) removes it in place and you do not advance i, because the next element slid into position i. When it is new, record it and step forward. The list shrinks under the loop, so len(seq) is re-checked each pass.

def dedupe(seq):
    seen = {}
    i = 0
    while i < len(seq):
        if seq[i] in seen:
            seq.pop(i)
        else:
            seen[seq[i]] = True
            i = i + 1
    return seq

Why it works. Each value's first occurrence is recorded and kept; every later occurrence is popped before i advances past it, so the surviving elements are exactly the first appearances, in order. Returning seq returns the same mutated object.

Complexity. Time O(n2) worst case, Space O(k) for k distinct values — each pop(i) shifts the tail, and the dictionary holds the kept values.

d1 = [1, 2, 2, 3, 1, 4]
assert dedupe(d1) == [1, 2, 3, 4]
assert d1 == [1, 2, 3, 4]
assert dedupe([]) == []
assert dedupe(['a', 'a', 'a']) == ['a']
d2 = [5, 5, 6, 5, 6, 7]
returned = dedupe(d2)
assert returned is d2
assert d2 == [5, 6, 7]

10. Playlist Object

Approach — dispatch dictionary holding state and behavior. The playlist is a dictionary that stores its four operations next to two pieces of state: the 'tracks' list and the 'index' of the now-playing song. Because each inner function closes over the name dispatch, they read and rewrite dispatch['tracks'] and dispatch['index'] directly — mutating dictionary entries, not rebinding local names, so no nonlocal is required. next advances the index with (index + 1) % len(tracks) so it wraps from the last track back to the first, and every function that indexes into tracks guards the empty case first. Each call to make_playlist builds its own dispatch dictionary, giving every playlist independent tracks and position.

def make_playlist():
    def add(title):
        dispatch['tracks'].append(title)
        return len(dispatch['tracks'])
    def current():
        tracks = dispatch['tracks']
        if not tracks:
            return None
        return tracks[dispatch['index']]
    def next_track():
        tracks = dispatch['tracks']
        if not tracks:
            return None
        dispatch['index'] = (dispatch['index'] + 1) % len(tracks)
        return tracks[dispatch['index']]
    def count():
        return len(dispatch['tracks'])
    dispatch = {'add': add,
                'current': current,
                'next': next_track,
                'count': count,
                'tracks': [],
                'index': 0}
    return dispatch

Why it works. Looking up p['add'] returns the stored function, and the next parentheses call it; inside, it appends to the 'tracks' list of the same dictionary the lookup came from, so the change is visible on every later access. The 'next' message is bound to next_track (the name next is a builtin, so the inner function is named differently while the dictionary key stays 'next'); its % keeps the index in range and cycles past the end. The empty-list guards make current and next return None before any out-of-range indexing.

Complexity. Time O(1) per operation, Space O(n) for n tracks — the dictionary holds four functions, the track list, and one index.

p = make_playlist()
assert p['count']() == 0
assert p['current']() is None
assert p['next']() is None
assert p['add']('Intro') == 1
assert p['add']('Verse') == 2
assert p['add']('Outro') == 3
assert p['current']() == 'Intro'
assert p['next']() == 'Verse'
assert p['next']() == 'Outro'
assert p['next']() == 'Intro'
assert p['count']() == 3
q = make_playlist()
assert q['add']('Solo') == 1
assert q['current']() == 'Solo'
assert p['count']() == 3
assert p['current']() == 'Intro'
