Installing Python on Windows

◷ approx. 4 min read

This guide takes you from a fresh Windows PC to a working python command, one step at a time. A single checkbox during installation prevents the most common beginner problem on Windows, so it is called out clearly below — if you read nothing else, read Step 3.

Step 1 — Open a terminal

On Windows, the plain text window where you type commands is PowerShell (Windows 11 bundles it inside an app called Terminal; either is fine). To open it:

  • Click the Start menu, type PowerShell, and press Enter.
  • (The older Command Prompt also works if you prefer it.)

A window appears with a line ending in > and a blinking cursor. That is the prompt — Windows saying "I'm ready; type a command." You type after it, never the prompt itself.

Step 2 — Check whether Python is already installed

At the prompt, type this and press Enter. (The > is the prompt Windows shows you — type only what comes after it, here python --version.)

> python --version
  • If you see Python 3.12.2 — any number beginning with 3 — Python is already installed. Skip ahead to Step 4.
  • If the Microsoft Store opens instead, or you see a message that Python was not found, it is not installed yet. (That Store pop-up is a Windows default we will sidestep.) Continue to Step 3.

Step 3 — Download and install Python

  1. Download. In your web browser, go to python.org/downloads. The page detects Windows and shows a large button to download the latest Python 3. Click it and wait for the installer (a .exe file) to finish downloading — it usually lands in your Downloads folder, and you can also click it in your browser's downloads bar.
  2. Tick the most important box. Open the downloaded file. On the very first installer screen, tick the checkbox labeled "Add python.exe to PATH" at the bottom before you click anything else. This one box is what lets you type python in any terminal afterward; skipping it is by far the most common Windows setup problem. (PATH is simply the list of places Windows looks when you type a command — ticking the box adds Python to that list.)
  3. Install. With the box ticked, click Install Now and let it run. At the end you may see an offer to "Disable path length limit" — accepting it is safe but not required. Close the installer.

Step 4 — Confirm it works

Close your PowerShell window and open a fresh one (Step 1) so it picks up the new install. Then check the version:

> python --version
Python 3.12.2

Any answer starting with 3 means you are ready. Now start Python itself by typing its name with nothing after it:

> python
>>>

The prompt changes to >>>, which means Python is now listening for code. Try one line:

>>> 2 + 2
4

If Python answers 4, your setup is complete. To leave Python and return to the ordinary > prompt, type quit() and press Enter (or press Control-Z then Enter — inside the terminal that means "done," not "undo").

> python vs python3. On Windows the command is just python, while the course text writes python3 (the macOS and Linux spelling). The python.org installer does not create a python3 command, so on Windows type python wherever the course says python3 — they refer to the same Python.

When something goes wrong

  • Typing python opens the Microsoft Store, or you get "not recognized." This is the classic sign that the Add to PATH box was not ticked during install. The fix: reopen the installer (or download it again), choose Uninstall, then run the installer once more so you get the original first screen — and this time tick "Add python.exe to PATH" before clicking Install Now. Then close and reopen PowerShell.
  • Still opening the Store after reinstalling. Windows keeps a shortcut that hijacks the word python. Turn it off: open Settings → Apps → Advanced app settings → App execution aliases, and switch off the entries named python.exe and python3.exe. Reopen PowerShell and try again.
  • python --version shows an old 2.x number. That is a leftover Python 2. Install the latest from python.org (Step 3); if both remain, use the newer one.
  • The terminal seems to "do nothing" after you type python. It probably worked — a >>> prompt means you are now inside Python. Type quit() and Enter to come back out.

Once python --version answers with a 3, you are done here — head back to 1.1 Getting Started and pick up where you left off.
