1.7 · Recursive Functions — Coding Practice

◷ approx. 12 min read · 30 worked examples

These problems build fluency with recursion on integers: finding the base case, making each call smaller, taking the recursive leap of faith, printing on the way down versus the way up, splitting work across mutually recursive helpers, and counting with tree recursion.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Easy

1. Count the Digits

Return how many digits a non-negative integer n has. Use recursion: peel off one digit per call with n // 10, and stop when a single digit is left.

Examples

Input:  count_digits(7)
Output: 1
Explanation: 7 is a single digit, the base case.

Input:  count_digits(18117)
Output: 5
Explanation: 18117 has five digits; each call drops the last one.

Input:  count_digits(0)
Output: 1
Explanation: 0 counts as one digit (0 < 10 fires the base case).

  • n is an integer with 0 <= n.
  • Must use recursion (no while loop, no str).

A number below 10 has exactly one digit; otherwise it has one more digit than n // 10.

8. Climb the Stairs

You climb a staircase of n steps, taking either 1 or 2 steps at a time. Return how many distinct ordered ways there are to reach the top. This is tree recursion: from n you either take a single step (leaving n - 1) or a double step (leaving n - 2), and the two counts add.

Examples

Input:  climb_stairs(3)
Output: 3
Explanation: 1+1+1, 1+2, and 2+1.

Input:  climb_stairs(5)
Output: 8
Explanation: eight distinct step sequences reach the top.

Input:  climb_stairs(0)
Output: 1
Explanation: there is exactly one way to stand at the top having taken no steps.

Constraints

  • n is an integer with 0 <= n.
  • Must use tree recursion (two recursive calls added together). No loops.

Hint

Split on the size of the next step: climb_stairs(n - 1) + climb_stairs(n - 2). Reaching exactly 0 is one success; overshooting below 0 is 0.

Hard

9. Partitions into Distinct Parts

Return the number of ways to write a positive integer n as a sum of distinct positive parts, each part no larger than m. Distinct means no part size repeats. For example, 6 has the partitions 6, 1 + 5, 2 + 4, and 1 + 2 + 3 — four of them. Use the same two-choice split as count_partitions, but because parts must be distinct, once you use a part of size m the next largest allowed part drops to m - 1.

Examples

Input:  count_distinct_partitions(6, 6)
Output: 4
Explanation: 6; 1+5; 2+4; 1+2+3.

Input:  count_distinct_partitions(5, 5)
Output: 3
Explanation: 5; 1+4; 2+3.

Input:  count_distinct_partitions(7, 4)
Output: 2
Explanation: 3+4 and 1+2+4 are the only ones with no part above 4.

Constraints

  • n and m are integers with 0 <= n and 0 <= m.
  • Must use tree recursion. Parts must be distinct (each size used at most once).

Starter

def count_distinct_partitions(n, m):
    pass
Hint

Either use a part of size m — then count count_distinct_partitions(n - m, m - 1) (the part is now spent) — or skip it and count count_distinct_partitions(n, m - 1). The base cases match count_partitions: n == 0 is 1, n < 0 is 0, m == 0 is 0.

10. Largest Digit

Return the largest digit of a non-negative integer n, computed recursively. Compare the last digit n % 10 against the largest digit of the rest of the number.

Examples

Input:  largest_digit(18117)
Output: 8
Explanation: the digits are 1, 8, 1, 1, 7; the biggest is 8.

Input:  largest_digit(9437184)
Output: 9
Explanation: 9 is the largest digit present.

Input:  largest_digit(120)
Output: 2
Explanation: among digits 1, 2, 0 the largest is 2.

Constraints

  • n is an integer with 0 <= n.
  • Must use recursion (no max over a list, no str).

Starter

def largest_digit(n):
    pass
Hint

A single digit is its own largest. Otherwise recurse on n // 10, then return whichever is bigger: that result or the last digit n % 10.

Solutions

1. Count the Digits

Approach — peel one digit, add one. This is the sum_digits skeleton from this section, but instead of adding the digit's value we add 1 for each digit. The base case n < 10 is a single-digit number, which contributes one digit; every other call strips the last digit with n // 10 and adds 1.

def count_digits(n):
    if n < 10:
        return 1
    else:
        return 1 + count_digits(n // 10)

Why it works. Every call removes exactly one digit, so the count grows by 1 per call until the recursion bottoms out on the leading single digit, which contributes its own 1.

Complexity. Time O(d), Space O(d) for d digits — one frame per digit.

assert count_digits(7) == 1
assert count_digits(18117) == 5
assert count_digits(9437184) == 7
assert count_digits(0) == 1

2. Sum to N

Approach — linear recursion with n == 0 base case. Mirror the factorial shape: the sum up to n is n plus the sum up to n - 1. Each call is strictly smaller, so it marches down to the base case n == 0, which returns 0.

def sum_to(n):
    if n == 0:
        return 0
    else:
        return n + sum_to(n - 1)

Why it works. By the recursive leap of faith, assume sum_to(n - 1) returns the sum 0..n-1; adding n gives the sum 0..n. The base case sum_to(0) == 0 anchors the induction.

Complexity. Time O(n), Space O(n) — one frame per value from n down to 0.

assert sum_to(0) == 0
assert sum_to(1) == 1
assert sum_to(5) == 15
assert sum_to(100) == 5050

3. Integer Power

Approach — repeated multiplication via recursion. The identity base ** exp = base * base ** (exp - 1) is already a recursive case, and base ** 0 = 1 is the base case. Each call lowers exp by one until it reaches 0.

def power(base, exp):
    if exp == 0:
        return 1
    else:
        return base * power(base, exp - 1)

Why it works. Assuming power(base, exp - 1) is base multiplied exp - 1 times, multiplying by one more base yields base multiplied exp times. The base case returns the empty product 1.

Complexity. Time O(exp), Space O(exp) — one frame per remaining exponent.

assert power(2, 0) == 1
assert power(2, 10) == 1024
assert power(5, 3) == 125
assert power(3, 4) == 81

4. Count Down, Then Up

Approach — recursive printing, sandwiched call. This is exactly the cascade pattern: a print before the recursive call runs while the stack grows (the descent), and a print after it runs while the stack unwinds (the ascent). The base case n == 0 prints once and never recurses, so 0 sits alone in the middle.

def count_down_up(n):
    print(n)
    if n > 0:
        count_down_up(n - 1)
        print(n)

Why it works. The first print(n) fires on every call as frames are created, producing n, n-1, ..., 0. The second print(n) is paused inside each multi-step frame and fires as frames return, producing 1, ..., n. The base case has no second print waiting after it.

Complexity. Time O(n), Space O(n) — n + 1 stacked frames at the deepest point.

# count_down_up prints as a side effect and returns None.
assert count_down_up(0) is None
assert count_down_up(3) is None

5. Product of the Digits

Approach — sum_digits shape with multiplication. Same peel-the-last-digit structure, but combine with * instead of +. The base case n < 10 returns the lone digit; otherwise multiply the last digit n % 10 by the product of the remaining digits n // 10.

def digit_product(n):
    if n < 10:
        return n
    else:
        return (n % 10) * digit_product(n // 10)

Why it works. Each call contributes exactly one digit factor; trusting the recursive call to return the product of the leading digits, multiplying by the last digit completes the product. A 0 digit zeroes the whole result, as it should.

Complexity. Time O(d), Space O(d) for d digits.

assert digit_product(7) == 7
assert digit_product(234) == 24
assert digit_product(405) == 0
assert digit_product(111) == 1
assert digit_product(99999) == 59049

6. Reverse a Number

Approach — accumulator helper. A nested helper carries the reversed value built so far in acc. On each step it appends the current last digit to acc with acc * 10 + n % 10 and recurses on n // 10; at the final single digit it folds that digit in and returns. This is the recursion-as-state idea: the accumulator replaces the explicit total an iterative version would keep.

def reverse_num(n):
    def helper(n, acc):
        if n < 10:
            return acc * 10 + n
        else:
            return helper(n // 10, acc * 10 + n % 10)
    return helper(n, 0)

Why it works. Reading digits from least significant to most and shifting acc left by one place each time reconstructs the number in reverse. Leading zeros vanish naturally because they become trailing positions that contribute nothing to an integer.

Complexity. Time O(d), Space O(d) for d digits — one helper frame per digit.

assert reverse_num(5) == 5
assert reverse_num(1234) == 4321
assert reverse_num(100) == 1
assert reverse_num(2013) == 3102
assert reverse_num(1000) == 1

7. Count Multiples of Three

Approach — mutual-style helper plus a counting recursion. is_multiple_of_3 answers a single yes/no question by subtracting 3 until the value drops below 3, where being a multiple means landing exactly on 0. The counter count_multiples_of_3 walks n down to 0, adding 1 whenever the current value passes the test.

def is_multiple_of_3(n):
    if n < 3:
        return n == 0
    else:
        return is_multiple_of_3(n - 3)

def count_multiples_of_3(n):
    if n == 0:
        return 0
    elif is_multiple_of_3(n):
        return 1 + count_multiples_of_3(n - 1)
    else:
        return count_multiples_of_3(n - 1)

Why it works. Subtracting 3 repeatedly reaches 0 exactly when n was divisible by 3, and otherwise lands on 1 or 2. The counter visits every integer from n down to 1 once, contributing 1 per multiple, so the total is the count of multiples in range.

Complexity. Time O(n2), Space O(n) — the counter makes n calls and each tests divisibility in O(n) subtractions.

assert count_multiples_of_3(0) == 0
assert count_multiples_of_3(3) == 1
assert count_multiples_of_3(10) == 3
assert count_multiples_of_3(30) == 10
assert is_multiple_of_3(27) is True
assert is_multiple_of_3(28) is False

8. Climb the Stairs

Approach — tree recursion on the next step. This is the Fibonacci branching shape with counting base cases. Every way to reach the top either starts with a single step (then there are climb_stairs(n - 1) ways) or a double step (then climb_stairs(n - 2) ways). The two groups are non-overlapping, so add them. Reaching exactly 0 is one completed climb; dropping below 0 overshot and counts as 0.

def climb_stairs(n):
    if n == 0:
        return 1
    elif n < 0:
        return 0
    else:
        return climb_stairs(n - 1) + climb_stairs(n - 2)

Why it works. Splitting on the first step's size partitions every climb into exactly one of two groups, and each branch counts its group recursively. The base cases distinguish a perfect landing (1) from an overshoot (0), so only valid step sequences are counted.

Complexity. Time O(ϕn) (exponential, like naive fib), Space O(n) for the deepest branch of the call tree.

assert climb_stairs(0) == 1
assert climb_stairs(1) == 1
assert climb_stairs(2) == 2
assert climb_stairs(3) == 3
assert climb_stairs(5) == 8
assert climb_stairs(10) == 89

9. Partitions into Distinct Parts

Approach — two-choice split with a spent part. This is count_partitions adapted so each part size is used at most once. The single decision is still "use a part of size m or not." If we use it, we subtract m and the next largest allowed part drops to m - 1 because m is now spent — count_distinct_partitions(n - m, m - 1). If we skip it, the largest allowed part is m - 1 — count_distinct_partitions(n, m - 1). The three base cases are identical to count_partitions.

def count_distinct_partitions(n, m):
    if n == 0:
        return 1
    elif n < 0:
        return 0
    elif m == 0:
        return 0
    else:
        return count_distinct_partitions(n - m, m - 1) + count_distinct_partitions(n, m - 1)

Why it works. Every distinct partition either contains m or does not, and the two branches cover those cases without overlap. Forcing the "use m" branch to drop to m - 1 guarantees no size repeats. n == 0 is checked first so an exact completion counts as one success even when m has also reached 0.

Complexity. Time O(n⋅m) distinct subproblems but exponential without memoization, Space O(n+m) for recursion depth.

assert count_distinct_partitions(6, 6) == 4
assert count_distinct_partitions(0, 5) == 1
assert count_distinct_partitions(5, 5) == 3
assert count_distinct_partitions(10, 10) == 10
assert count_distinct_partitions(7, 4) == 2

10. Largest Digit

Approach — recurse, then compare the last digit. A single digit is its own largest. Otherwise compute the largest digit of the rest of the number (n // 10), then return whichever is bigger: that result or the current last digit n % 10. The comparison happens on the way back up, after the recursive call returns.

def largest_digit(n):
    if n < 10:
        return n
    else:
        rest = largest_digit(n // 10)
        last = n % 10
        if last > rest:
            return last
        else:
            return rest

Why it works. Taking the recursive leap of faith, largest_digit(n // 10) is the largest digit among all but the last. The maximum digit of the whole number is therefore the larger of that value and the last digit, which the conditional returns.

Complexity. Time O(d), Space O(d) for d digits — one frame per digit.

assert largest_digit(7) == 7
assert largest_digit(18117) == 8
assert largest_digit(9437184) == 9
assert largest_digit(120) == 2
assert largest_digit(54321) == 5
assert largest_digit(1000000) == 1
