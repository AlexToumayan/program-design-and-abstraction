2.6 · Implementing Classes and Objects — Coding Practice

◷ approx. 12 min read · 31 worked examples

These problems build the object system from the inside: instances as dispatch dictionaries with private attribute stores, classes that share behavior, the local-first attribute lookup rule, method binding that slips self in for you, instance attributes that shadow class attributes, and inheritance through a base class. Most problems are posed and solved on the hand-built kernel (make_instance, make_class, bind_method, init_instance) from the lesson, shown in the Setup block below.

> Work each problem from the starter signature first. Run your function against the Examples, then open the matching solution below to check your approach and its time and space cost.

Easy

1. A Box With a Default

Build a dispatch dictionary that hides a private store behind 'get' and 'set' messages. A 'get' for a name that was never 'set' should return a fixed default supplied when the box is made. Two boxes must not share state.

Examples

Input:  box = make_box(0); box['set']('width', 12); box['get']('width')
Output: 12
Explanation: a stored value is returned from the private store.

Input:  box = make_box(0); box['get']('height')
Output: 0
Explanation: 'height' was never set, so the default fills in.

Input:  make_box('?')['get']('width')
Output: '?'
Explanation: each box closes over its own store and its own default.

Constraints

  • Return a dictionary with exactly the keys 'get' and 'set'.
  • The store must be private: nothing outside make_box may reach it except through the two messages.
  • Do not use classes or the class statement.

Hint

Define get and set inside make_box so both close over the same private contents dictionary; get checks membership before falling back to default.

2. Local-First Lookup

Write the attribute-lookup rule by hand. Given a dictionary of instance attributes, a dictionary of class attributes, and a name, return the instance's value if it has one, otherwise the class's value, otherwise None.

Examples

Input:  lookup({'balance': 20}, {'interest': 0.02}, 'balance')
Output: 20
Explanation: found on the instance, so the class is never consulted.

Input:  lookup({'balance': 20}, {'interest': 0.02}, 'interest')
Output: 0.02
Explanation: missing locally, so it falls back to the class.

Input:  lookup({'balance': 20}, {'interest': 0.02}, 'holder')
Output: None
Explanation: found in neither dictionary.

Constraints

  • Check the instance dictionary first, then the class dictionary.
  • An instance entry must shadow a class entry of the same name.
  • Use the in membership test, not exceptions.

Hint

This is the folder-then-handbook rule: if name in instance_attrs ... elif name in class_attrs ... else None.

Input:  bound = bind(greet, {'get': lambda name: 'Ada'}); bound('!')

Explanation: greet(self, punct) ran with self supplied automatically.

Explanation: a number is not callable, so it passes straight through.

Explanation: binding a function yields a callable wrapper.

  • Use the built-in callable to decide whether to bind.
  • The wrapper must accept any number of trailing arguments and place instance first.

Solutions

Setup

Every solution builds on the object-system kernel from the 2.6 lessons: instances and classes are dispatch dictionaries, attribute lookup is local-first, methods are bound so the instance becomes self, and a class may fall back to a base class. Define these four functions once before running any solution below.

1. A Box With a Default

Approach — a dispatch dictionary over a private store. This is the warm-up make_box from the lesson with one twist: get falls back to a default instead of failing. Both get and set close over the same private contents dictionary, so the store survives after make_box returns and is reachable only through the two messages.

def make_box(default):
    contents = {}

    def get(name):
        if name in contents:
            return contents[name]
        else:
            return default

    def set(name, value):
        contents[name] = value

    return {'get': get, 'set': set}

Why it works. Each call to make_box creates a fresh frame with its own contents and default, and the returned closures capture that frame, so different boxes never share state. The in test makes a stored value shadow the default.

Complexity. Time O(1) per message, Space O(k) for k stored keys. <Dictionary lookups and inserts are constant time.>

box = make_box(0)
assert box['get']('width') == 0          # missing keys fall back to the default
box['set']('width', 12)
assert box['get']('width') == 12         # a stored value wins over the default
assert box['get']('height') == 0         # height was never set
box2 = make_box('?')
assert box2['get']('width') == '?'       # a fresh box has its own private store
assert box['get']('width') == 12         # the first box is undisturbed

2. Local-First Lookup

Approach — folder, then handbook. This is exactly the rule make_instance uses, written out plainly: check the instance dictionary first, fall back to the class dictionary, and yield None when neither has the name.

def lookup(instance_attrs, class_attrs, name):
    if name in instance_attrs:
        return instance_attrs[name]
    elif name in class_attrs:
        return class_attrs[name]
    else:
        return None

Why it works. Because the instance branch is tested first, an instance entry of the same name shadows the class entry — the same shadowing the real system gives you for free.

Complexity. Time O(1), Space O(1). <Two membership tests and one read.>

inst = {'balance': 20}
cls = {'interest': 0.02, 'bank': 'Aurora'}
assert lookup(inst, cls, 'balance') == 20      # found on the instance
assert lookup(inst, cls, 'interest') == 0.02   # missing locally, found on class
assert lookup(inst, cls, 'bank') == 'Aurora'   # missing locally, found on class
assert lookup(inst, cls, 'holder') is None     # found nowhere
inst2 = {'balance': 5, 'interest': 0.09}
assert lookup(inst2, cls, 'interest') == 0.09  # instance shadows the class

3. Bind a Method by Hand

Approach — wedge in self. This is bind_method rebuilt. A callable is wrapped in an inner method that calls it with instance first and the caller's arguments spread after; a non-callable is returned untouched because plain data has no self.

def bind(fn, instance):
    if callable(fn):
        def method(*args):
            return fn(instance, *args)
        return method
    else:
        return fn

Why it works. The first star in def method(*args) gathers every argument the caller passes into a tuple; the second star in fn(instance, *args) spreads that tuple back out after instance, so a function written as (self, punct) is callable with just punct.

Complexity. Time O(1) to bind, Space O(1). <One closure is created.>

def greet(self, punct):
    return self['get']('name') + punct

obj = {'get': lambda name: 'Ada'}
bound = bind(greet, obj)
assert bound('!') == 'Ada!'                # self was slipped in for us
assert bind(0.02, obj) == 0.02             # non-callable values pass through
assert bind('hello', obj) == 'hello'       # strings are not callable
assert callable(bind(greet, obj))          # binding a function yields a callable

4. A One-Attribute Class

Approach — locals() into make_class. Bind the shared attribute and the method in the factory's frame, then hand that frame to make_class. The instance has no local label, so reads fall back to the class; show is bound to the instance and reads through the same fallback.

def make_label_class(text):
    label = text

    def show(self):
        return self['get']('label')

    return make_class(locals())

Why it works. locals() snapshots {'label': text, 'show': <function>}, so the class dictionary shares one label across all its instances. Each call to make_label_class builds a new frame, so the two classes are independent.

Complexity. Time O(1) per lookup, Space O(1). <One class dictionary with two entries.>

Tag = make_label_class('SALE')
assert Tag['get']('label') == 'SALE'           # class attribute read by message
t = Tag['new']()                               # no __init__, so new() takes no args
assert t['get']('label') == 'SALE'             # instance falls back to the class
assert t['get']('show')() == 'SALE'            # bound method reads the label
Other = make_label_class('NEW')
assert Other['get']('label') == 'NEW'          # a separate class, its own snapshot
assert Tag['get']('label') == 'SALE'           # original class untouched

5. A Thermostat Class

Approach — instance state through self['set']. The constructor files target into the instance's private store; unit stays a shared class attribute. Each method reads target, computes the new value, writes it back, and returns it — so state persists per instance.

def make_thermostat_class():
    unit = 'C'

    def __init__(self, target):
        self['set']('target', target)

    def warmer(self, amount):
        new_target = self['get']('target') + amount
        self['set']('target', new_target)
        return self['get']('target')

    def cooler(self, amount):
        new_target = self['get']('target') - amount
        self['set']('target', new_target)
        return self['get']('target')

    return make_class(locals())

Why it works. Because target lives in each instance's own attributes dictionary (written by set_value), two thermostats never collide; unit is read by fallback from the one shared class dictionary.

Complexity. Time O(1) per method call, Space O(1) per instance. <Each instance stores one number.>

Thermostat = make_thermostat_class()
room = Thermostat['new'](20)
assert room['get']('target') == 20             # set by __init__
assert room['get']('unit') == 'C'              # inherited from the class
assert room['get']('warmer')(3) == 23          # bound method updates state
assert room['get']('cooler')(5) == 18          # state carried between calls
assert room['get']('warmer')(2) == 20
office = Thermostat['new'](16)
assert office['get']('target') == 16           # a second instance is independent
assert room['get']('target') == 20             # the first instance is unaffected

6. Shadow a Class Attribute

Approach — set shadows, never edits. rate is a class attribute reached by fallback. When one instance does ['set']('rate', 25), set_value stores rate in that instance's own dictionary; because charge reads rate through self['get'], the local value now wins on that instance while the class and other instances are untouched.

def make_membership_class():
    rate = 10

    def __init__(self, name):
        self['set']('name', name)

    def charge(self, months):
        return self['get']('rate') * months

    return make_class(locals())

Why it works. A set message always stores on the receiver's own dictionary. The instance's rate therefore shadows the class's rate for that one object, exactly mirroring simple.interest = 0.04 in built-in Python.

Complexity. Time O(1) per call, Space O(1) per shadowing instance. <One extra key when an instance overrides.>

Membership = make_membership_class()
basic = Membership['new']('Kirk')
premium = Membership['new']('Spock')
assert basic['get']('charge')(3) == 30         # uses the class rate of 10
premium['set']('rate', 25)                     # store a local override
assert premium['get']('rate') == 25            # instance value wins
assert Membership['get']('rate') == 10         # class is untouched
assert basic['get']('rate') == 10             # other instances still see the class
assert premium['get']('charge')(3) == 75       # the override flows into the method
assert basic['get']('charge')(3) == 30         # basic still bills at 10/month

7. A Method That Counts Its Calls

Approach — read-modify-write on instance state. count lives in each instance's private store. push reads the old count, writes old + 1, and returns it; total only reads. Each instance has its own count, so two turnstiles tally independently.

def make_turnstile_class():
    def __init__(self):
        self['set']('count', 0)

    def push(self):
        self['set']('count', self['get']('count') + 1)
        return self['get']('count')

    def total(self):
        return self['get']('count')

    return make_class(locals())

Why it works. State changes happen through self['set'], which updates only the receiving instance's attributes dictionary, so the counters never interfere.

Complexity. Time O(1) per call, Space O(1) per instance. <Each instance stores one integer.>

Turnstile = make_turnstile_class()
gate = Turnstile['new']()
assert gate['get']('total')() == 0             # starts at zero
assert gate['get']('push')() == 1
assert gate['get']('push')() == 2
assert gate['get']('push')() == 3
assert gate['get']('total')() == 3             # total reflects the pushes
gate2 = Turnstile['new']()
assert gate2['get']('push')() == 1             # independent counter
assert gate['get']('total')() == 3             # first gate unaffected

8. A Subclass Default

Approach — inheritance, not mutation. make_class(attrs, base) builds a class whose own dictionary holds just the new attribute and whose fallback parent is base. Any name the subclass lacks — including __init__ and describe — is resolved by climbing to base, so the base class is never modified.

def make_plain_class():
    def __init__(self, color):
        self['set']('color', color)

    def describe(self):
        return self['get']('color') + ' ' + self['get']('shape')

    return make_class(locals())

def add_default(base, name, value):
    attrs = {name: value}
    return make_class(attrs, base)

Why it works. describe reads shape through self['get']: it misses the instance, then finds shape in the subclass's own attrs. The names color and __init__ are absent from the subclass, so lookup falls back to Plain. The base class's dictionary gains nothing, so Plain['get']('shape') stays None.

Complexity. Time O(d) for inheritance depth d per lookup, Space O(1) for the new class. <Each miss walks one link up the chain.>

Plain = make_plain_class()
Shaped = add_default(Plain, 'shape', 'circle')
item = Shaped['new']('red')
assert item['get']('color') == 'red'           # __init__ inherited from Plain
assert item['get']('shape') == 'circle'        # supplied by the subclass default
assert item['get']('describe')() == 'red circle'
assert Plain['get']('shape') is None           # the base class is unchanged
other = Shaped['new']('blue')
assert other['get']('describe')() == 'blue circle'

9. A Subclass That Overrides One Method

Approach — override and delegate to the base. FloorTimer defines only tick; remaining and __init__ are inherited from Timer. The override calls the base's tick directly through base['get']('tick')(self, amount) — unbound, with self passed explicitly — then clamps a negative result to 0. Asking the base class (not self) avoids infinite recursion into the override.

def make_floor_timer_class(base):
    def tick(self, amount):
        result = base['get']('tick')(self, amount)
        if result < 0:
            self['set']('seconds', 0)
        return self['get']('seconds')

    return make_class(locals(), base)

Why it works. base['get']('tick') retrieves the original, unbound base method, so self must be supplied by hand. The base method already wrote seconds; the override only corrects it when it went negative. The un-subclassed Timer keeps its raw behavior.

Complexity. Time O(1) per call, Space O(1) per instance. <One subtraction and at most one clamp.>

def make_timer_class():
    def __init__(self, seconds):
        self['set']('seconds', seconds)

    def remaining(self):
        return self['get']('seconds')

    def tick(self, amount):
        self['set']('seconds', self['get']('seconds') - amount)
        return self['get']('seconds')

    return make_class(locals())

Timer = make_timer_class()
FloorTimer = make_floor_timer_class(Timer)

t = FloorTimer['new'](10)
assert t['get']('remaining')() == 10           # remaining is inherited from Timer
assert t['get']('tick')(4) == 6                # override delegates to base tick
assert t['get']('tick')(4) == 2
assert t['get']('tick')(5) == 0                # would be -3, but the floor clamps it
assert t['get']('tick')(1) == 0               # stays clamped at zero
plain = Timer['new'](3)
assert plain['get']('tick')(5) == -2           # the base class still goes negative

10. A Frozen Instance Wrapper

Approach — a dispatch dictionary that forwards reads and swallows writes. freeze returns a new object whose get forwards to the real instance (so inherited and bound methods still work) and whose set ignores its arguments, changes nothing, and returns 'frozen'. The underlying instance is left fully mutable.

def freeze(instance):
    def get_value(name):
        return instance['get'](name)

    def set_value(name, value):
        return 'frozen'

    return {'get': get_value, 'set': set_value}

Why it works. The wrapper is itself a dispatch dictionary with the same 'get'/'set' shape. Its get_value simply delegates to the underlying instance, so every read — including a fetched bound method — passes through to the real object, while set_value is a no-op gate that returns 'frozen'. Mutating the raw instance directly still works, and the wrapper reflects those changes on the next read.

Complexity. Time O(1) per message, Space O(1). <One forwarding closure pair.>

def make_point_class():
    def __init__(self, x, y):
        self['set']('x', x)
        self['set']('y', y)

    def move(self, dx, dy):
        self['set']('x', self['get']('x') + dx)
        self['set']('y', self['get']('y') + dy)
        return (self['get']('x'), self['get']('y'))

    def coords(self):
        return (self['get']('x'), self['get']('y'))

    return make_class(locals())

Point = make_point_class()

p = Point['new'](1, 2)
frozen = freeze(p)
assert frozen['get']('coords')() == (1, 2)     # reads still work through the wrapper
assert frozen['set']('x', 99) == 'frozen'      # writes are refused
assert frozen['get']('x') == 1                 # the value never changed
assert p['get']('x') == 1                      # the underlying instance is safe
assert p['get']('move')(3, 4) == (4, 6)        # the raw instance is still mutable
assert frozen['get']('coords')() == (4, 6)     # the wrapper reflects raw changes
