Chapter 2/2.3 Sequences

2.3 Lists and Sequence Iteration

◷ approx. 16 min read · 25 worked examples · best read in two passes

So far you have given names to single values and built small machines (functions) that take a few inputs. This lesson is where you learn to handle many values at once without inventing a fresh name for every one of them. By the end you will be able to do five things: build a list and read its parts, ask a list for its length and pull out an element by position, combine and search lists with sequence operations, walk through every element with a for statement, and generate a run of integers with range.

The thread that ties all of this together is one idea: a sequence is an ordered collection of values. Ordered is the load-bearing word. A sequence is not a loose bag of items where order does not matter; every element sits at a known position, and you can ask for "the element at position 2" and get a definite answer.

Sequences Are Ordered Collections

Think about a numbered to-do list stuck to your fridge.

1. eggs
2. milk
3. rice

Two facts make this a sequence rather than a pile. First, it has a definite length: there are exactly three items, and you can count them. Second, it has element selection: each item lives at a position, so "the second item" unambiguously means milk. Those two properties — a finite length, and an element for every position — are exactly what the textbook uses to define a sequence in general, and every sequence type in Python shares them.

Python sequences work just like the fridge list, with one twist that trips up nearly every beginner: Python counts positions starting at 0, not 1. So the eggs would be at position 0, the milk at position 1, and the rice at position 2. We will come back to why zero is a sensible starting point, because the reason makes the rest of indexing feel natural instead of arbitrary.

Lists Are Sequences You Write With Square Brackets

The most common sequence you will build is a list. A list can hold any values, in any order, and it can be as long as you like. You write one by placing its elements between square brackets, separated by commas.

# (1)
digits = [1, 8, 2, 8]

After line (1) runs, the name digits refers to a list of four numbers. A useful way to picture it is a row of labeled boxes, with the position index written above each box:

name: digits

index:   0   1   2   3
       +---+---+---+---+
value: | 1 | 8 | 2 | 8 |
       +---+---+---+---+

The name digits is not the list itself. It is a label that points to the list value, exactly the way a name pointed to a number in the previous chapter. Several names could point to this same list, and the picture would not change — only the labels hanging off it would.

Follow the Line: A List Literal

The right-hand side of (1) is a list literal — the actual brackets-and-commas notation that builds a fresh list. Read it left to right and notice what each piece is doing, not just what it is named:

[1, 8, 2, 8]
├─ [ ........... opens a new list literal
├─ 1 .......... element 0: the value placed in the first position
├─ , ........... separates one element from the next
├─ 8 .......... element 1: the value placed in the second position
├─ , ........... separates one element from the next
├─ 2 .......... element 2: the value placed in the third position
├─ , ........... separates one element from the next
├─ 8 .......... element 3: the value placed in the fourth position
├─ ] ........... closes the list literal
└─ evaluates to  the list [1, 8, 2, 8]

The commas are what tell Python where one element ends and the next begins. The spaces after the commas are purely for human eyes — Python does not require them — but we write them anyway, because readable code is how you catch your own mistakes before the machine does.

Length and Element Selection

Two operations work on every sequence, and you saw both in the definition above. The built-in function len reports how many elements a sequence contains.

# (2)
len(digits)

Follow the line:

len(digits)
├─ len ......... the built-in function: counts the elements of a sequence
├─ ( ........... begins the input
├─ digits ...... the sequence to measure
├─ ) ........... ends the input
└─ evaluates to  4

So the interpreter displays:

4

The second operation is element selection: square brackets written immediately after a sequence, with an index inside, hand you back the element at that position.

# (3)
digits[3]

Follow the line:

digits[3]
├─ digits ...... the sequence to select from
├─ [ ........... begins an element selection
├─ 3 ........... the index: which position to read (counting from 0)
├─ ] ........... ends the selection
└─ evaluates to  8

Index 3 is the fourth box, because counting starts at 0. The interpreter displays:

8

Here is the official transcript from the text, which introduces a list, measures it, and selects from it in one short session:

>>> digits = [1, 8, 2, 8]
>>> len(digits)
4
>>> digits[3]
8

Why Indexing Starts at Zero

The cleanest way to make zero-based indexing feel right is to read an index as "how many steps from the front?" rather than "which number item."

index 0: take 0 steps from the front
index 1: take 1 step from the front
index 2: take 2 steps from the front
index 3: take 3 steps from the front

Line up those steps against the list:

[1, 8, 2, 8]
 ^
 front (0 steps lands here)

Taking zero steps leaves you standing on the first element, 1. Taking three steps lands you on 8, the last element. Under this reading the last valid index is always one less than the length: a four-element list has length 4 and a final index of 3. Hold onto that fact — the most common list error in the language comes from forgetting it.

Combining Lists: Concatenation and Repetition

Lists support some of the same operators you have seen on numbers, but the meaning is adapted to sequences. The + operator concatenates two lists, joining them end to end into a brand-new list.

# (4)
[2, 7] + digits

If digits is [1, 8, 2, 8], then (4) produces a six-element list:

[2, 7, 1, 8, 2, 8]

Notice that + here does not add numbers position-by-position. It glues the second list onto the back of the first. Follow the line to see the two operands and the result:

[2, 7] + digits
├─ [2, 7] ...... the left list (two elements)
├─ + ........... the concatenation operator for sequences
├─ digits ...... the right list, currently [1, 8, 2, 8]
└─ evaluates to  [2, 7, 1, 8, 2, 8]

The * operator repeats a list. Given a list and an integer k, it returns a new list made of k copies of the original, laid end to end.

# (5)
digits * 2

This produces:

[1, 8, 2, 8, 1, 8, 2, 8]

Read * carefully: it does not multiply each element by 2. It repeats the whole list twice. The result is a new eight-element list; the original digits is untouched.

Now the official example, which combines both operators in a single expression:

# (6)
[2, 7] + digits * 2

Operator precedence matters here, and it matches arithmetic: * binds tighter than +, so Python evaluates digits * 2 first, then concatenates [2, 7] onto the front. Follow the evaluation:

[2, 7] + digits * 2
├─ digits * 2 ........ evaluated first: [1, 8, 2, 8, 1, 8, 2, 8]
├─ [2, 7] + (...) .... then concatenate [2, 7] onto the front
└─ evaluates to       [2, 7, 1, 8, 2, 8, 1, 8, 2, 8]

The official transcript:

>>> [2, 7] + digits * 2
[2, 7, 1, 8, 2, 8, 1, 8, 2, 8]

Membership: Asking Whether a Value Is Inside

You will often want to ask a yes-or-no question: does this sequence contain a particular value? The in operator answers exactly that, returning True or False.

# (7)
2 in digits
5 in digits

For digits = [1, 8, 2, 8], the results are:

True
False

Follow the line on the first one to keep the three pieces straight:

2 in digits
├─ 2 ........... the value to search for
├─ in .......... the membership operator: is it present?
├─ digits ...... the sequence to search through
└─ evaluates to  True

Under the hood, Python walks the sequence and compares each element to the target, stopping the moment it finds a match (or returning False once it runs off the end). 2 appears at index 2, so the first line is True; 5 never appears, so the second is False.

Nested Lists: Lists Inside Lists

Because a list can hold any values, it can hold other lists. This is what lets you represent grids, pairs, rows of data, and trees later on.

# (8)
pairs = [[10, 20], [30, 40]]

The outer list has exactly two elements, and each element is itself a two-element list:

index:   0          1
       +----------+----------+
value: | [10, 20] | [30, 40] |
       +----------+----------+

Selecting from the outer list gives you back an inner list, not a number:

# (9)
pairs[1]

Follow the line:

pairs[1]
├─ pairs ....... the outer list
├─ [1] ......... select index 1: the second element
└─ evaluates to  [30, 40]

To reach a number, select again on the result. Element selection can be chained, applied as many times as the nesting is deep:

# (10)
pairs[1][0]

Python evaluates this left to right: it does the first selection, then applies the second selection to whatever that produced.

pairs[1][0]
├─ pairs[1] .... select index 1 of the outer list -> [30, 40]
├─ [0] ......... then select index 0 of that inner list
└─ evaluates to  30

Spelled out as two steps:

Step 1: pairs[1]      ->  [30, 40]
Step 2: [30, 40][0]   ->  30

The official transcript:

>>> pairs = [[10, 20], [30, 40]]
>>> pairs[1]
[30, 40]
>>> pairs[1][0]
30

Sequence Iteration: Doing Something With Every Element

The real payoff of sequences is that you can process all of their elements with a few lines of code, no matter how long the sequence is. Consider a concrete task: count how many times a particular value appears in a sequence.

Before reaching for a new tool, you can already solve this with a while loop and an explicit index counter. The textbook writes it exactly this way first:

# (11)
def count(s, value):
    """Count the number of occurrences of value in sequence s."""
    total, index = 0, 0
    while index < len(s):
        if s[index] == value:
            total = total + 1
        index = index + 1
    return total

Read what (11) is doing. It keeps two running numbers: total, the count so far, and index, the position it is currently inspecting. The while loop runs as long as index is a valid position (index < len(s)). Each pass looks at s[index], bumps total when it matches value, and then advances index by one so the next pass looks at the next element. When index reaches the length, every position has been checked and the loop stops.

That works, but notice the bookkeeping: you have to create index, compare it against the length, select s[index] by hand, and remember to increment it. Forgetting that last index = index + 1 would loop forever. The for statement removes all of that ceremony by handing you the elements directly.

# (12)
def count(s, value):
    """Count the number of occurrences of value in sequence s."""
    total = 0
    for elem in s:
        if elem == value:
            total = total + 1
    return total

Version (12) does the same job as (11), but there is no index, no length check, and no manual increment. The for statement walks the sequence for you, and on each pass the name elem is bound to the next element. You compare elem directly to value. This is the version you should reach for whenever you need every element and do not need its position.

Both definitions answer the canonical example the same way:

# (13)
count(digits, 8)

Since 8 appears at index 1 and again at index 3, the result is:

2

The official transcript pairs the for-loop definition with this call:

>>> def count(s, value):
        """Count the number of occurrences of value in sequence s."""
        total = 0
        for elem in s:
            if elem == value:
                total = total + 1
        return total

>>> count(digits, 8)
2

Follow the Line: The For Statement Header

The new piece of syntax is the header line of the loop. Focus on it by itself:

for elem in s:
├─ for ......... begin a loop that visits each element of a sequence
├─ elem ........ the loop name: rebound to the current element each pass
├─ in .......... connects the loop name to the sequence being traversed
├─ s ........... the sequence expression to iterate over
├─ : ........... begins the indented body (the suite) that repeats
└─ does .......  runs the suite once for every element, in order

This is a statement, not an expression, so the final line states the action it performs rather than a value it evaluates to: it runs the indented suite once per element. The indented lines after the colon are the body, and indentation is what physically marks them as "inside the loop."

# (14)
for elem in s:
    if elem == value:
        total = total + 1

If a line is indented under the for, it repeats once per element. If a line is not indented under it, the loop does not touch it. We will see in the traps section how getting that indentation wrong silently breaks a function.

The General Form of a For Statement

The textbook gives the shape of every for statement as a template. The pieces in angle brackets are placeholders you fill in — they are not literal Python you type:

for <name> in <expression>:
    <suite>

And it spells out the exact procedure Python follows when it executes one:

1. Evaluate the header <expression>, which must yield an iterable value (something you can walk through element by element, such as a list).
2. For each element value in that iterable value, in order:
     a. Bind <name> to that value in the current frame.
     b. Execute the <suite>.

Three things in that procedure are worth underlining. The <expression> must be iterable — a list and a range both qualify. The <name> is rebound on every pass, not created fresh each time; it simply points to a new element. And the suite runs in order, first element to last, which is why order is part of what makes a sequence a sequence.

Trace: Counting Step by Step

Tracing a loop by hand is the fastest way to trust it. Trace this call against the for-loop version (12):

# (15)
count([1, 8, 2, 8], 8)

When the call starts, a frame for count is created with the arguments bound and total initialized:

count frame
+-------+--------------+
| s     | [1, 8, 2, 8] |
| value | 8            |
| total | 0            |
+-------+--------------+

Now the for statement runs the suite once per element, rebinding elem each pass:

Pass 1:
  elem = 1
  elem == value  ->  1 == 8  ->  False
  total stays 0

Pass 2:
  elem = 8
  elem == value  ->  8 == 8  ->  True
  total becomes 1

Pass 3:
  elem = 2
  elem == value  ->  2 == 8  ->  False
  total stays 1

Pass 4:
  elem = 8
  elem == value  ->  8 == 8  ->  True
  total becomes 2

Loop ends (no elements left).
return total  ->  2

The list [1, 8, 2, 8] is never copied or split apart. The same list is traversed once, and elem points to one element at a time as the loop advances.

Sequence Unpacking: Naming Several Parts at Once

A common shape is a sequence whose elements are themselves short, fixed-length sequences — a list of pairs, for instance. A for statement can list several names in its header to pull each element apart into its pieces in one move. This is called sequence unpacking.

Start with a verifiable bridge before the official example. Suppose you want to add up the second number of every pair:

On each pass Python expects the current element to hold exactly two values, and it binds them to first and second in order:

Pass 1:  [1, 10]  ->  first = 1, second = 10  ->  running = 10
Pass 2:  [2, 20]  ->  first = 2, second = 20  ->  running = 30
Pass 3:  [3, 30]  ->  first = 3, second = 30  ->  running = 60

You can confirm by hand that running ends at 60. Now the official example, which counts how many pairs have two equal elements:

The header for x, y in pairs: unpacks each inner list into x and y:

Pass 1:  [1, 2]  ->  x = 1, y = 2  ->  1 == 2 is False, same_count stays 0
Pass 2:  [2, 2]  ->  x = 2, y = 2  ->  2 == 2 is True,  same_count becomes 1
Pass 3:  [2, 3]  ->  x = 2, y = 3  ->  2 == 3 is False, same_count stays 1
Pass 4:  [4, 4]  ->  x = 4, y = 4  ->  4 == 4 is True,  same_count becomes 2

The final value of same_count is 2, because [2, 2] and [4, 4] are the pairs whose two elements are equal. The official transcript:

>>> pairs = [[1, 2], [2, 2], [2, 3], [4, 4]]
>>> same_count = 0
>>> for x, y in pairs:
        if x == y:
            same_count = same_count + 1

>>> same_count
2

Ranges: Sequences of Integers

Often you do not have a list to walk; you just want a loop to run a certain number of times, or to step through a run of integers. The range type is a built-in sequence of integers built for exactly this. You create one with range, which in its common two-argument form takes a start and an end.

A subtle and important point comes first: a range is a sequence, but it is not a list. When you ask the REPL for a range directly, it shows you the range itself, not its contents.

# (18)
range(1, 10)

The official transcript makes the boundary behavior explicit with a comment — the start is included, the end is not:

>>> range(1, 10)  # Includes 1, but not 10
range(1, 10)

To actually see the integers a range stands for, convert it to a list with list.

# (19)
list(range(5, 8))

Follow the line:

list(range(5, 8))
├─ range(5, 8) ... the integers from 5 up to but not including 8
├─ list(...) ..... collect those integers into an actual list
└─ evaluates to    [5, 6, 7]

The result:

[5, 6, 7]

If you give range a single argument, it is treated as the end, and the start defaults to 0:

# (20)
list(range(4))

This produces:

[0, 1, 2, 3]

The official transcript shows all three together:

>>> range(1, 10)  # Includes 1, but not 10
range(1, 10)
>>> list(range(5, 8))
[5, 6, 7]
>>> list(range(4))
[0, 1, 2, 3]

A range with n as its single argument therefore produces the n integers 0 through n - 1 — exactly the valid indices of a length-n sequence, which is no accident.

Running a Loop a Fixed Number of Times

The most frequent use of range is as the expression in a for header, to repeat a suite a set number of times. When the loop name is not actually needed inside the body, Python programmers conventionally use the single underscore _ as the name.

This runs the body three times, printing the same line each time:

Go Bears!
Go Bears!
Go Bears!

The underscore is still an ordinary name — there is nothing magic about it. By convention it signals "I have to bind something here, but I do not plan to use the value." The official transcript:

>>> for _ in range(3):
        print('Go Bears!')

Go Bears!
Go Bears!
Go Bears!

⚠Common Beginner Traps

The first trap is using the length as if it were the last index. The valid indices of a length-4 list are 0 through 3, so reaching for index 4 runs off the end:

# (22)
digits = [1, 8, 2, 8]
digits[4]

This raises an IndexError, because there is no element at position 4. The fix is the rule from earlier: the last index is len(s) - 1.

The second trap is reading digits * 2 as "double every element." It does not touch the elements; it repeats the whole list, giving [1, 8, 2, 8, 1, 8, 2, 8]. Likewise + between two lists concatenates them rather than adding matching positions.

The third trap is putting return inside the loop when it belongs after it:

# (23)
def broken_count(s, value):
    total = 0
    for elem in s:
        if elem == value:
            total = total + 1
        return total

Because return total is indented under the for, the function returns on the very first pass, after looking at only one element. The indentation, not your intention, decides what is inside the loop. Moving return total out one level (so it lines up with total = 0) fixes it.

The fourth trap is unpacking an element into the wrong number of names:

# (24)
for x, y in [[1, 2], [3, 4, 5]]:
    print(x, y)

The first element [1, 2] unpacks cleanly into x and y, but the second element [3, 4, 5] has three values and cannot fit into exactly two names, so Python raises a ValueError mid-loop. Sequence unpacking requires every element to have exactly as many parts as you list names.

The fifth trap is expecting a range to print its integers. Typing range(5, 8) at the prompt shows range(5, 8), not [5, 6, 7]; you must wrap it in list(...) to see the contents.

✓Check Your Understanding

  1. For digits = [1, 8, 2, 8], what is the value of digits[0], and why?
  2. Why does digits[4] raise an error for that same list?
  3. What does [1, 2] + [3, 4] produce, and how is that different from adding numbers?
  4. In for elem in s:, at what moment does elem change, and what does it change to?
  5. Using the for-loop version of count from example (12), what does count([4, 4, 4, 7], 4) return?
  6. What does list(range(4)) produce, and what does plain range(1, 10) display in the REPL?
Show the answers
  1. digits[0] is 1. Indexing starts at 0, so index 0 is the first element — zero steps from the front.
  2. The list has four elements, so its only valid indices are 0, 1, 2, and 3. Index 4 is one slot past the end, which raises an IndexError.
  3. It produces [1, 2, 3, 4]. The + operator on lists concatenates them end to end; it does not add elements at matching positions the way + on two numbers would.
  4. elem changes at the start of every pass through the loop. Python takes the next element of s, in order, and rebinds elem to that element for the duration of that one pass.
  5. It returns 3. The for loop binds elem to each element in turn; the value 4 appears at the first three positions and the comparison elem == value is True each of those times, bumping total to 3, while the final 7 leaves it unchanged.
  6. list(range(4)) produces [0, 1, 2, 3]. Plain range(1, 10) displays range(1, 10), because a range shows itself rather than its contents until you convert it with list.

§Summary

A sequence is an ordered collection of values, defined by two properties: a finite length and element selection by index, with indices starting at 0. The most common sequence is a list, written with square brackets and commas. len reports how many elements a sequence has; square brackets with an index select an element; + concatenates lists and * repeats one; in tests membership; and lists can be nested so that selection chains like pairs[1][0] reach deeply nested values. The for statement walks a sequence, rebinding a loop name to each element in order and running its suite once per element — replacing the manual index bookkeeping of a while loop. A for header can unpack fixed-length elements into several names at once. Finally, range generates a sequence of integers (start included, end excluded), most often used to run a loop a fixed number of times. Together these let you write one short piece of code that processes any number of values.
