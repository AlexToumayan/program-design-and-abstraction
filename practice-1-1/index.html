Practice/Chapter 1 · Coding Practice

1.1 · Getting Started — Coding Practice

◷ approx. 6 min read · 20 worked examples

These problems build the day-one reading skill of Section 1.1: predicting the value or the exact printed output of an expression before you run it. You will separate code into tokens, trace nested call expressions inside out, tell true division (/) apart from floor division (//), and watch names get bound and rebound in the global frame.

> Section 1.1 comes before you learn to define your own functions (that is Section 1.3), so there is nothing to "write" here. Instead, each problem shows an expression or a short program. Predict its value (or its exact printed output) by hand, then open the matching solution below to check your trace — and run the confirm block to be sure.

Easy

1. Division Makes a Float

Predict the value Python displays for this expression. The two numbers divide evenly — does that change the kind of number you get back?

Expression

9 / 3
Hint

The / operator always asks for true division, and true division always produces a float — even when the result is a whole number.

2. Two Kinds of Division

The same pair of numbers is divided two ways. Predict the value of each expression, and note which is an int and which is a float.

Expression

7 / 2
7 // 2
Hint

/ is true division (keeps the fractional part, gives a float); // is floor division (drops the fractional part, gives an int here).

3. Distance With abs

abs is a built-in function that returns the size of a number, ignoring its sign. Predict the value of this call expression by computing the subtraction inside the parentheses first.

Expression

abs(4 - 9)
Hint

Python must finish the argument 4 - 9 before it can call abs, so compute that difference first, then take its absolute value.

4. Smallest of Three

min returns the smallest of the values you hand it, and it accepts more than two. Predict the value of this call.

Expression

min(12, 5, 8)
Hint

The commas separate three distinct arguments; min compares all of them and returns the least.

Medium

5. Powers as a Call

pow is a built-in function whose first argument is the base and whose second is the exponent, so pow(base, exponent) computes base raised to that power. Predict the value of this call.

Expression

pow(2, 5)
Hint

Read it as "2 multiplied by itself 5 times," i.e. 2 * 2 * 2 * 2 * 2.

6. A Call Wrapping Two Calls

Predict the value of this expression. The outer max cannot run until both of its arguments are finished values, so evaluate the two inner calls first.

Expression

max(min(10, 4), abs(-7))
Hint

Compute min(10, 4) and abs(-7) first, then hand both results to max.

7. Floor Division Goes Down

Floor division rounds down toward negative infinity, which is not the same as chopping off the fraction toward zero. Predict the value of this expression and watch the sign.

Expression

-7 // 2
Hint

-7 / 2 is -3.5; flooring means stepping to the next integer below -3.5, not toward zero.

8. A Binding Is a Snapshot

This is a short program, not a single expression. After it runs, predict the value of total8 and the value of g8.

Expression

g8 = 6
total8 = g8 * 4
g8 = 100
Hint

Assignment evaluates the right side now and stores that finished number. Rebinding g8 later does not recompute total8.

Hard

9. Stacks, Leftovers, and a Rebinding

Predict the exact printed output of this program — every line, in order. Three ideas appear at once: floor division, true division, and a name that is rebound partway through.

Expression

Hint

stacks and leftover are both computed while coins is 47; the later coins = stacks only changes what coins means from then on, so it does not touch the already-stored stacks or leftover.

10. A Tree of Calls Inside print

Predict the exact printed output of this program. Build the expression tree for the argument to print and evaluate it from the leaves up — three separate calls feed into one max.

Expression

print(max(abs(-3), min(10, 6), 18 // 4))
Hint

Finish abs(-3), min(10, 6), and 18 // 4 first; then max compares those three results and print displays the winner.

Solutions

1. Division Makes a Float

Approach — / always yields a float. As Section 1.1 shows with 1 / 2, the / symbol asks for true division, and true division returns a floating-point number. Even when the division comes out even, the result keeps its decimal point.

Trace.

9 / 3                -> 3.0   (a float, because / is true division)

Value. 3.0

Run it to confirm:

print(9 / 3)

3.0

2. Two Kinds of Division

Approach — true vs floor division. / keeps the fractional part and returns a float; // discards the fractional part (flooring the result) and, for two ints, returns an int. Same operands, two different operators, two different values.

Trace.

7 / 2                -> 3.5   (true division, a float)
7 // 2               -> 3     (floor division, an int)

Values. 7 / 2 is 3.5; 7 // 2 is 3.

Run it to confirm:

print(7 / 2)
print(7 // 2)

3.5
3

3. Distance With abs

Approach — argument first, then call. A call expression cannot apply its function until the argument is a finished value, so Python evaluates 4 - 9 before calling abs. 4 - 9 is -5, and abs(-5) is 5.

Trace.

4 - 9                -> -5
abs(-5)              -> 5

Value. 5

Run it to confirm:

print(abs(4 - 9))

5

4. Smallest of Three

Approach — one call, three arguments. Just as max returns the largest input in Section 1.1, min returns the smallest. The commas keep 12, 5, and 8 as three separate arguments, and min returns the least of them.

Trace.

min(12, 5, 8)        -> 5

Value. 5

Run it to confirm:

print(min(12, 5, 8))

5

5. Powers as a Call

Approach — a two-argument call. pow has the same call shape as max: function_name(argument_1, argument_2). The first argument is the base, the second the exponent, so pow(2, 5) is 2 raised to the 5th power.

Trace.

pow(2, 5)            -> 2 * 2 * 2 * 2 * 2 = 32

Value. 32

Run it to confirm:

print(pow(2, 5))

32

6. A Call Wrapping Two Calls

Approach — evaluate operands first. The two inner calls are the arguments of max, and a call cannot run until its arguments are finished values. So Python computes min(10, 4) and abs(-7) before the outer max.

Trace.

min(10, 4)           -> 4
abs(-7)              -> 7
max(4, 7)            -> 7

Value. 7

Run it to confirm:

print(max(min(10, 4), abs(-7)))

7

7. Floor Division Goes Down

Approach — floor means round toward negative infinity. -7 / 2 is -3.5. Floor division does not chop toward zero; it takes the largest integer that is less than or equal to the true quotient. The integer just below -3.5 is -4.

Trace.

-7 / 2  (true)       -> -3.5
-7 // 2 (floor)      -> floor(-3.5) = -4

Value. -4

Run it to confirm:

print(-7 // 2)

-4

8. A Binding Is a Snapshot

Approach — a name stores the value computed now. Assignment evaluates the right side, then binds the name to that finished value; it does not store a live formula. So total8 keeps the number it was given even after g8 is rebound.

Trace.

g8 = 6               Global frame: g8 -> 6
total8 = g8 * 4      look up g8 (6), compute 24; total8 -> 24
g8 = 100             rebind only g8; total8 is untouched

Values. total8 is 24; g8 is 100.

Run it to confirm:

24
100

9. Stacks, Leftovers, and a Rebinding

Approach — three ideas at once. coins // 10 floors the division to count full stacks (4). coins / 10 is true division, giving the float 4.7. Both are computed while coins is 47. Then coins = stacks rebinds coins to 4, which does not change the already-stored stacks or leftover.

Trace.

coins = 47
stacks = coins // 10    47 // 10 -> 4
leftover = coins / 10   47 / 10  -> 4.7  (true division, a float)
coins = stacks          rebind coins -> 4
print(stacks)           displays 4
print(leftover)         displays 4.7
print(coins)            displays 4

Output.

4
4.7
4

Run it to confirm:

4
4.7
4

10. A Tree of Calls Inside print

Approach — leaves up. Draw the expression tree for print's single argument and let values percolate upward. The three operands of max are themselves finished computations, so evaluate each one, then max over the three, then print the result.

Trace.

print(max(abs(-3), min(10, 6), 18 // 4))
└── max(abs(-3), min(10, 6), 18 // 4)
    ├── abs(-3)     -> 3
    ├── min(10, 6)  -> 6
    └── 18 // 4     -> 4

max(3, 6, 4)        -> 6
print(6)            displays 6

Output.

6

Run it to confirm:

print(max(abs(-3), min(10, 6), 18 // 4))

6
